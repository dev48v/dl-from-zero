DeepLearningFromZero · Day 38

🪛 Stochastic Depth — a 100-layer ResNet is slow to train and its gradients still fade. So randomly drop whole residual blocks each training pass: each block gets a survival probability (deeper blocks drop more, on a linear schedule); a dropped block becomes a plain identity bypass, so the network is shallower that step. At test time you keep every block but scale its output by its survival probability. It’s dropout’s drop-a-whole-LAYER cousin — trains very deep nets faster and regularises them as an implicit ensemble of many shorter networks

A tower of residual blocks. Flip a coin on each one — the survivors do the work, the rest are bypassed.

The green line down the left is the ResNet identity highway — it always carries the signal (and a clean +1 gradient) straight through. Each block adds a residual branch F(x) on top. In training, every block draws a coin from its own survival probability p: heads it stays (alive, computes x + F(x)), tails it’s dropped to a bare identity. With linear decay the deeper blocks are dropped more often. Hit Sample a training pass to roll every coin at once — watch the network get shorter, and the gradient’s trip get quicker. At test time nothing is dropped: every block stays, but its branch is scaled by p so the expected signal matches training.

🔨 Build it yourself

    
The live demo is a from-scratch stochastic-depth stack — no library, just a survival schedule, a coin flip per block, and an identity bypass. We build it one line at a time: a residual block with an identity skip, the linearly-decaying survival probabilities, the training forward pass that drops blocks with a Bernoulli coin, the full stack, the expected-depth calculation, the test-time forward that scales instead of drops — then we map it to the couple of lines you actually write with PyTorch / timm.

    
    
      
  1. 
        
    1
    A residual block — the identity skip is what makes dropping safe
    
        
    Every block is y = x + F(x): an identity shortcut plus a learned branch F (conv→norm→relu→conv). Because the shortcut is already there, deleting F leaves a perfectly valid, still-connected network — the block just becomes a pass-through. That is the whole reason you can drop a layer at all.
    
        
    // one residual block: identity highway + a residual branch F
function block(x, F) {
  return add(x, F(x));      // y = x + F(x)
}
// drop F entirely and you still have a valid net:  y = x  (identity)
    
      
    2. 
      
  2. 
        
    2
    Survival probabilities — deeper blocks drop more (linear decay)
    
        
    Each block l of L gets a keep-chance p_l. The paper’s rule is a straight line: the first block survives almost always, and it decays linearly to p_L (e.g. 0.5) at the deepest block. Early features are precious, so protect them; late blocks are redundant, so drop them freely.
    
        
    // linear decay:  p_1 ~ 1  ->  p_L at the last block
function survival(l, L, pL) {
  return 1 - (l / L) * (1 - pL);   // l = 1..L
}
// pL = 0.5, L = 54  ->  p_1 = 0.99 ... p_54 = 0.50
// uniform variant: every block just uses the same pL
    
      
    3. 
      
  3. 
        
    3
    Training forward — a Bernoulli coin per block
    
        
    On each training step, draw an independent coin for every block. Heads (prob p_l): run the block normally, x + F(x). Tails: skip the branch completely and pass x straight through. The dropped branch does no compute and gets no gradient this step.
    
        
    function trainBlock(x, F, p) {
  if (Math.random() < p) return add(x, F(x));  // survives: x + F(x)
  else                    return x;             // dropped: identity
}
// note: NOT scaled during training in the original paper —
// the survival happens or it doesn't (a hard 0/1 gate).
    
      
    4. 
      
  4. 
        
    4
    The whole stack, one training pass
    
        
    Chain the blocks: feed x through each, dropping some. Whatever survives forms a shorter sub-network for this step — a different one every time. The identity highway guarantees the signal (and gradient) still reaches every end even when many blocks are gone.
    
        
    function forwardTrain(x, blocks, L, pL) {
  blocks.forEach((F, i) => {
    const p = survival(i + 1, L, pL);
    x = trainBlock(x, F, p);       // some blocks become identity
  });
  return x;                        // effective depth varies each step
}
    
      
    5. 
      
  5. 
        
    5
    Expected depth — how deep the net really is on average
    
        
    The expected number of active blocks is just the sum of the survival probabilities. With linear decay to 0.5 over 54 blocks it’s about 40 — so you train a ~54-block net at the cost of a ~40-block one, and the gradients only have to survive ~40 transforms on average.
    
        
    function expectedDepth(L, pL) {
  let e = 0;
  for (let l = 1; l <= L; l++) e += survival(l, L, pL);
  return e;                         // E[active blocks] = Σ p_l
}
// L=54, pL=0.5  ->  ~40 active blocks on average (~25% shorter)
    
      
    6. 
      
  6. 
        
    6
    Test time — keep every block, scale by survival
    
        
    At inference you want the full, deterministic network — no coins. So keep all blocks but weight each branch by its survival probability: y = x + p_l·F(x). That makes the expected contribution match what training saw. It’s exactly dropout’s train-drop / test-scale trick, moved up from neurons to whole layers.
    
        
    function testBlock(x, F, p) {
  return add(x, scale(F(x), p));    // y = x + p * F(x)  (no dropping)
}
// every block runs; its branch is damped by p so E[output] matches
// training. Modern frameworks often calibrate this automatically.

  7. 
        
    7
    Why it works — an ensemble of shorter nets + a gradient shortcut
    
        
    Two effects. (1) Every step’s surviving subset is a different network, so training implicitly averages over an exponential family of shallower nets — strong regularisation, like an ensemble. (2) A shorter active path means backprop passes through fewer nonlinearities, so gradients stay healthy and training is faster — the original paper trained a 1202-layer ResNet that a plain net couldn’t.
    
        
    // implicit ensemble: 2^L possible sub-networks, one sampled per step
// gradient path this step = number of ALIVE blocks (not L)
//   fewer nonlinearities to backprop through -> less vanishing
// net effect: deeper nets train FASTER and generalise BETTER
//   (CIFAR-10: a 1202-layer ResNet trainable only with this)

  8. 
        
    8
    The real thing — a few lines in PyTorch / timm
    
        
    You never hand-roll the coin. Torchvision ships StochasticDepth and timm calls it DropPath — you drop it inside each residual block and hand it the per-block drop rate (1−p, decayed with depth). It’s a no-op at eval() (the module rescales for you), needs no architecture change, and is standard in ResNets, EfficientNet, and every modern ViT / ConvNeXt.
    
        
    from torchvision.ops import StochasticDepth   # timm: from timm.models.layers import DropPath

class ResBlock(nn.Module):
    def __init__(self, ch, drop_prob):
        super().__init__()
        self.F = nn.Sequential(conv(ch), norm(ch), relu(), conv(ch))
        self.drop = StochasticDepth(drop_prob, mode="row")  # drop_prob = 1 - p_l

    def forward(self, x):
        return x + self.drop(self.F(x))       # train: drops whole branch; eval: scales

# per-block drop rate, linearly increasing with depth:
drop_rates = [i / (L - 1) * 0.5 for i in range(L)]   # 0.0 ... 0.5
# family of "drop things to generalise" tricks: dropout, dropconnect, drop-path.

🎉 Day 38 of DeepLearningFromZero done.
Stochastic Depth takes dropout’s “drop things to generalise” idea from single neurons up to whole residual blocks. Each block gets a survival probability that decays linearly with depth (early blocks kept, deep blocks dropped often); on every training step a Bernoulli coin either runs the block (x + F(x)) or collapses it to a bare identity bypass — safe precisely because a ResNet block already carries an identity skip. The network is therefore shorter that step, so backprop crosses fewer nonlinearities (healthier gradients, faster training) and, across steps, the model averages over an exponential implicit ensemble of shallower sub-networks (regularisation). At test time you drop nothing and instead scale each branch by its survival p, so expected activations match training — the same train-drop / test-scale bargain as dropout. The payoff: you can train absurdly deep nets (the paper’s 1202-layer ResNet), get a solid accuracy bump for free, and it’s one line (StochasticDepth / DropPath) in every modern ResNet, EfficientNet and ViT. 👉 Tomorrow, Day 39: DenseNet — instead of dropping connections, add them: every layer feeds directly into every later layer, maximal feature reuse.
