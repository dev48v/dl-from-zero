🧠 Multi-Head Attention — one attention is a single lens; run h of them in parallel over different subspaces, then concatenate and project

One softmax can only look one way at a time. Multi-head attention runs several in parallel.

Each token embedding is projected to a query, key and value, then those are split into h heads of width dk=d/h. Every head runs the real scaled dot-product attention softmax(QKᴛ/√dk)V in its own subspace, so different heads can specialise in different relations. 🔨 Build it yourself

    
You saw that a single attention is one relation lens, and that splitting Q/K/V into h heads lets the model read several relations at once for the same cost. Here it is in code: scaled dot-product attention for one head, the Q/K/V projections, the reshape that splits d into h×dk, all heads attending in parallel, the concat + output projection, and finally the whole thing as a module — then the one-liner nn.MultiheadAttention, plus the MQA/GQA variants modern LLMs use to shrink the KV-cache.

    
    
      
  1. 
        
    1
    Scaled dot-product attention (one head)
    
        
    The atom everything is built from: score every query against every key with a dot product, divide by √dk, softmax into weights that sum to 1, and take that weighted average of the values.
    
        
    import numpy as np

def softmax(x, axis=-1):
    x = x - x.max(axis, keepdims=True)
    e = np.exp(x)
    return e / e.sum(axis, keepdims=True)

def attention(Q, K, V):                 # Q:(t,dk) K:(s,dk) V:(s,dv)
    dk = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(dk)       # (t,s)  -- the scaling matters
    W = softmax(scores, axis=-1)         # each row sums to 1
    return W @ V, W

out, W = attention(np.random.randn(4,2), np.random.randn(4,2), np.random.randn(4,2))
print(W.sum(1))                          # [1. 1. 1. 1.]

    Project the tokens into Q, K, V
    
        
    Each token embedding x (width d) is linearly mapped to a query, a key and a value by three learned matrices. Query = “what am I looking for”, key = “what do I offer”, value = “what I pass on”.
    
        
    import numpy as np
d = 8
X = np.random.randn(7, d)               # 7 tokens, width d
Wq, Wk, Wv = (np.random.randn(d, d) for _ in range(3))

Q = X @ Wq                              # (7, d)
K = X @ Wk
V = X @ Wv
# all three are still width d -- we split that width across heads next

    Split the width into h heads
    
        
    Multi-head attention is a reshape, not new parameters. Cut the width d into h contiguous chunks of dk=d/h. Each chunk is one head’s subspace. In PyTorch you reshape (B,T,d) → (B,T,h,dk) and move the head axis up front.
    
        
    import torch
B, T, d, h = 1, 7, 8, 4
dk = d // h                             # 2

Q = torch.randn(B, T, d)
def split_heads(x):                     # (B,T,d) -> (B,h,T,dk)
    B, T, d = x.shape
    return x.view(B, T, h, dk).transpose(1, 2)

Qh = split_heads(Q)
print(Qh.shape)                         # torch.Size([1, 4, 7, 2])
assert h * dk == d                      # the split is exact

    All heads attend in parallel
    
        
    With the head axis batched, one set of matrix multiplies runs every head at once. Each head has its own dk-dim scores and its own softmax — that independence is what lets them specialise.
    
        
    import torch, torch.nn.functional as F

def mha_scores(Qh, Kh, Vh):             # each (B,h,T,dk)
    dk = Qh.shape[-1]
    scores = Qh @ Kh.transpose(-2, -1) / dk ** 0.5   # (B,h,T,T)
    W = F.softmax(scores, dim=-1)                     # per-head, per-row
    return W @ Vh, W                                  # (B,h,T,dk)

ctx, W = mha_scores(Qh, Kh, Vh)
print(W.sum(-1))                        # all ones: every head's rows sum to 1

    Concatenate heads, then project with Wₒ
    
        
    Glue the h head outputs (dk each) back into one width-d vector, then pass through a final learned matrix WO that lets information mix across heads. Without WO the heads would never talk to each other.
    
        
    def merge_heads(x):                     # (B,h,T,dk) -> (B,T,d)
    B, h, T, dk = x.shape
    return x.transpose(1, 2).contiguous().view(B, T, h * dk)

Wo = torch.randn(d, d)
out = merge_heads(ctx) @ Wo             # (B,T,d) -- back to model width
# concat width == d_model, ready for the residual add + LayerNorm

    The whole module — and the one-liner
    
        
    Wrapped as a layer, and then the batteries-included version. nn.MultiheadAttention does exactly the six steps above. The comment shows MQA/GQA: share K/V across heads to shrink the KV-cache at inference — the trick behind fast modern LLM decoding.
    
        
    import torch, torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d, h):
        super().__init__()
        self.h, self.dk = h, d // h
        self.Wq = nn.Linear(d, d); self.Wk = nn.Linear(d, d)
        self.Wv = nn.Linear(d, d); self.Wo = nn.Linear(d, d)
    def forward(self, x):               # x: (B,T,d)
        B, T, d = x.shape
        sh = lambda t: t.view(B, T, self.h, self.dk).transpose(1, 2)
        q, k, v = sh(self.Wq(x)), sh(self.Wk(x)), sh(self.Wv(x))
        a = (q @ k.transpose(-2,-1) / self.dk**0.5).softmax(-1)
        o = (a @ v).transpose(1,2).reshape(B, T, d)
        return self.Wo(o)

# batteries included -- identical math:
mha = nn.MultiheadAttention(embed_dim=8, num_heads=4, batch_first=True)
y, attn_w = mha(x, x, x)                # self-attention

# MQA (num_kv_heads=1) / GQA (a few kv heads) share K,V across query heads
#   -> smaller KV-cache, faster decoding, used by Llama-2 70B, Mistral, ...

🎉 Day 30 of DeepLearningFromZero done.
A single attention is one lens: one softmax over one dot-product space, so it can only track one kind of relation at a time. Multi-head attention projects each token to Q/K/V, splits the width d into h heads of dk=d/h, and runs softmax(QKᴛ/√dk)V independently in each subspace, then concatenates the heads back to width d and mixes them with an output projection WO. The √dk keeps the dot products from saturating softmax as dimensions grow. Crucially the cost is identical to one big head — the same four d×d matrices, just reshaped — yet the model gains the ability to read adjacency, syntax, word-identity and delimiter/sink relations all at once, as you saw in the four heatmaps. Empirically trained heads really do split this way, and many are redundant enough to prune; MQA/GQA share K/V across heads to shrink the inference KV-cache. 👉 Tomorrow, Day 31: RMSNorm — the leaner LayerNorm (drop the mean-centering, keep only the scale) that Llama and most modern transformers now use.
