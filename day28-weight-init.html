⚖️ Weight Initialization — why the first random numbers decide whether a deep net learns at all

Before a network learns anything, it has to guess its weights. That guess alone can kill training.

Below is a real forward pass through a 10-layer deep net (64 units per layer), fed a fixed random input with std 1. Pick an init scheme and an activation, press Run, and watch the standard deviation of the activations as the signal travels deeper. On a log axis the story is instant: all-zeros makes every unit identical (symmetry never breaks, a dead network); too-small (×0.01) shrinks the signal geometrically until it vanishes; too-large saturates tanh at ±1 or explodes ReLU toward overflow; Xavier/Glorot holds the std flat under tanh; and He holds it flat under ReLU. Each run leaves a faint ghost line so you can stack the schemes and see the contrast. The std at every layer, and the histogram on the right, are computed live from the actual pass.

A pre-activation is a sum of fan_in weight×input products, so Var(z) = fan_in · Var(w) · Var(x). Xavier and He simply choose Var(w) so that fan_in · Var(w) = 1 (Xavier balances the backward pass too; He doubles it because ReLU zeroes half its inputs). 🔨 Build it yourself

    
You just watched a real 10-layer forward pass live or die on nothing but its starting weights: zeros freeze it by symmetry, tiny weights vanish the signal, big weights saturate or explode it, and the two variance-preserving schemes hold the std flat with depth. Here it is in code: see the collapse in raw NumPy, derive Xavier and He by hand, then call the one-liners PyTorch ships — nn.init.xavier_normal_ and nn.init.kaiming_normal_ — and apply them across a whole model.

    See the collapse: a naive forward pass
    
        
    This is exactly what the LOOK demo runs. Scale the weights too small and the activation std decays toward 0 layer by layer; scale them too big and it blows up. Print the per-layer std and watch it drift.
    
        
    import numpy as np
np.random.seed(0)

N, L = 64, 10
x = np.random.randn(N)                 # input, std = 1

for scale in [0.01, 0.20, 1.00]:       # too-small, ~ok, too-large
    a = x.copy()
    stds = []
    for _ in range(L):
        W = np.random.randn(N, N) * scale   # naive fixed scale
        a = np.tanh(W @ a)                  # one layer
        stds.append(a.std())
    print(f"scale={scale:>4}: layer1={stds[0]:.2e}  layer10={stds[-1]:.2e}")
# scale=0.01 -> layer10 ~ 0      (vanished)
# scale=1.00 -> layer10 saturates/blows up depending on activation

    Xavier / Glorot by hand (tanh, sigmoid)
    
        
    A pre-activation is a sum of fan_in products, so Var(z) = fan_in · Var(w) · Var(x). To keep Var(z) = Var(x) on the forward pass you need Var(w) = 1/fan_in; balancing the backward pass too gives Glorot's compromise Var(w) = 2/(fan_in+fan_out). Near 0 tanh has slope 1, so the variance carries through unchanged.
    
        
    import numpy as np

def xavier_normal(fan_in, fan_out):
    std = np.sqrt(2.0 / (fan_in + fan_out))   # Glorot: balances fwd + bwd
    return np.random.randn(fan_out, fan_in) * std

def xavier_uniform(fan_in, fan_out):
    limit = np.sqrt(6.0 / (fan_in + fan_out)) # same variance, uniform form
    return np.random.uniform(-limit, limit, (fan_out, fan_in))

W = xavier_normal(64, 64)   # for a 64 -> 64 tanh/sigmoid layer
# feed std-1 input through many such layers -> std stays ~1

    He / Kaiming by hand (ReLU) — and why the 2
    
        
    ReLU sets every negative pre-activation to 0, which roughly halves the variance passing through. So you must double the forward term: Var(w) = 2/fan_in. That extra factor of 2 is the entire difference between He and Xavier, and it is what keeps a deep ReLU stack from slowly vanishing.
    
        
    import numpy as np

def he_normal(fan_in, fan_out):
    std = np.sqrt(2.0 / fan_in)   # the 2 undoes ReLU zeroing half the inputs
    return np.random.randn(fan_out, fan_in) * std

def he_uniform(fan_in, fan_out):
    limit = np.sqrt(6.0 / fan_in)
    return np.random.uniform(-limit, limit, (fan_out, fan_in))

W = he_normal(64, 64)   # for a 64 -> 64 ReLU layer
# Xavier here would under-scale and let the ReLU signal slowly decay

    PyTorch one-liners: xavier_ and kaiming_
    
        
    You never hand-roll this in practice. torch.nn.init ships both, in normal and uniform variants. Note the trailing underscore: these fill the tensor in place. Biases go to 0 — the weights already break symmetry.
    
        
    import torch, torch.nn as nn

lin = nn.Linear(64, 64)

# tanh / sigmoid layer -> Xavier / Glorot
nn.init.xavier_normal_(lin.weight)                 # or xavier_uniform_
# ReLU layer -> He / Kaiming (tell it the nonlinearity + which fan)
nn.init.kaiming_normal_(lin.weight,
                        mode="fan_in", nonlinearity="relu")  # or kaiming_uniform_
nn.init.zeros_(lin.bias)                            # biases start at 0

    Apply it to a whole model
    
        
    model.apply(fn) walks every submodule so you can initialise a deep stack in one shot. Match the scheme to the activation: He for ReLU/GELU nets, Xavier for tanh/sigmoid and attention projections. The gain from calculate_gain adjusts for the exact nonlinearity.
    
        
    import torch.nn as nn

def init_weights(m):
    if isinstance(m, nn.Linear):
        nn.init.kaiming_normal_(m.weight, nonlinearity="relu")  # ReLU net
        if m.bias is not None:
            nn.init.zeros_(m.bias)

model = nn.Sequential(nn.Linear(64,64), nn.ReLU(),
                      nn.Linear(64,64), nn.ReLU(),
                      nn.Linear(64,10))
model.apply(init_weights)          # every Linear re-initialised in place

# tanh gain, if you need it explicitly:
g = nn.init.calculate_gain("tanh") # -> ~1.667; std = g * sqrt(2/(fi+fo))

    Init still matters even with BatchNorm / LayerNorm
    
        
    Normalization (Days 19 & 27) re-centers activations every layer, so it makes a network far less sensitive to the starting weights — but not immune. The very first optimisation steps, any block without a norm, and very deep residual stacks still need sane init; modern nets even scale residual branches down (e.g. 1/√(2L)) on top of He / Xavier.
    
        
    import torch.nn as nn

# LayerNorm / BatchNorm params get their own standard init:
ln = nn.LayerNorm(64)
nn.init.ones_(ln.weight)    # gamma = 1
nn.init.zeros_(ln.bias)     # beta  = 0   (identity at start)

# Deep residual nets often down-scale the residual branch on top of He:
#   out = x + (1/sqrt(2*num_layers)) * sublayer(x)
# Norm reduces init sensitivity; it does NOT remove the need for good init.

🎉 Day 28 of DeepLearningFromZero done.
You saw that a deep network's fate is sealed before the first gradient step: all-zeros traps every unit in the same value by symmetry, too-small weights vanish the signal, too-large weights saturate tanh or explode ReLU, and only variance-preserving init keeps the activation std flat with depth. The rule is one line of algebra — a pre-activation is a sum of fan_in products, so choose Var(w) to cancel that fan_in factor: Xavier/Glorot 2/(fan_in+fan_out) for tanh/sigmoid, and He/Kaiming 2/fan_in for ReLU, the factor of 2 paying for the half of the inputs ReLU zeroes.
