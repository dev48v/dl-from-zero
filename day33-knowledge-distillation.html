🎓 Knowledge Distillation — a tiny student network learns to copy a big teacher not by its final answer but by its whole softened probability distribution — the "dark knowledge" of how wrong each wrong class is — and ends up nearly as accurate for a fraction of the cost

One image of a cat. Watch the teacher's confident answer melt into a soft distribution — and watch the student learn far more from that melt than from a bare label.

The teacher is a big trained network; for this cat photo it outputs raw scores (logits) over five classes. A one-hot label says only "cat = 1, everything else = 0" and throws away everything else. But the teacher knows more: it thinks this cat looks a bit like a dog, slightly like a tiger, and nothing like a car or plane. That inter-class structure is the dark knowledge. To expose it we divide the logits by a temperature T before softmax: at T=1 the distribution is peaky (cat ≈ 1), and as T rises it softens, lifting the runner-ups into view. The student is then trained to match this soft distribution — loss = α·CE(hard label) + (1−α)·T²·KL(teacher-soft ‖ student). Turn T and α, then press Train student and watch a fresh 5-weight student slide onto the teacher's curve.

🔨 Build it yourself

    
Distillation adds almost nothing to a normal training loop: freeze a trained teacher, run it in eval() mode to get soft targets, and swap the plain cross-entropy for a weighted sum of the hard-label loss and a soft-label loss. Here it is from scratch first, so every term is visible, then the idiomatic PyTorch version with KLDivLoss, then the full loop, temperature/alpha tips, and the self-distillation variant.

    
    
      
  1. 
        
    1
    Temperature softmax — the only new primitive
    
        
    Divide the logits by T before softmax. T=1 is ordinary softmax (peaky). Larger T flattens the distribution and lifts the small probabilities — the dark knowledge — into a range where they actually contribute to the loss.
    
        
    import torch, torch.nn.functional as F

def soft_probs(logits, T):
    return F.softmax(logits / T, dim=-1)     # T>1 -> softer, higher entropy

z = torch.tensor([4.0, 1.8, 1.0, -1.5, -2.5])   # teacher: a cat image
print(soft_probs(z, T=1))   # ~[.86, .095, .043, .003, .001]  (peaky)
print(soft_probs(z, T=4))   # ~[.42, .24, .19, .09, .06]      (dark knowledge visible)
print(soft_probs(z, T=100)) # -> nearly uniform (all ~0.2)

    The distillation loss from scratch
    
        
    Two terms. The soft term is the KL divergence from the teacher's soft distribution to the student's, computed at temperature T and multiplied by T² (the soft gradients scale as 1/T², so this keeps them on the same footing as the hard term). The hard term is ordinary cross-entropy against the true label at T=1. Blend with α.
    
        
    def distill_loss(student_logits, teacher_logits, target, T=4.0, alpha=0.3):
    # soft: match the teacher's softened distribution
    p_teacher = F.softmax(teacher_logits / T, dim=-1)
    log_q_student = F.log_softmax(student_logits / T, dim=-1)
    kl = F.kl_div(log_q_student, p_teacher, reduction="batchmean")
    soft = (T * T) * kl                       # <-- the T^2 rescale

    # hard: match the ground-truth label (standard CE, T=1)
    hard = F.cross_entropy(student_logits, target)

    return alpha * hard + (1 - alpha) * soft

    The training loop — freeze the teacher, train the student
    
        
    The teacher is loaded, set to eval(), and wrapped in torch.no_grad() — it produces targets but never learns. Only the (much smaller) student's parameters get gradients. Everything else is a normal loop.
    
        
    teacher = load_pretrained().eval()            # big, frozen
student = SmallNet()                          # e.g. 1/10 the params
opt = torch.optim.AdamW(student.parameters(), lr=3e-4)

for x, y in loader:
    with torch.no_grad():                     # teacher never updates
        t_logits = teacher(x)
    s_logits = student(x)
    loss = distill_loss(s_logits, t_logits, y, T=4.0, alpha=0.3)
    opt.zero_grad(); loss.backward(); opt.step()

    Choosing T and α
    
        
    These are the two knobs that matter. T controls how much dark knowledge you expose (too low = back to one-hot; too high = the signal drowns toward uniform). α balances trusting the true label vs trusting the teacher. Typical starting points below.
    
        
    # common defaults (Hinton et al. 2015, and follow-ups)
T     = 4.0      # try 2-6; higher for a very confident teacher
alpha = 0.1      # 0.1-0.5: mostly soft targets, a little hard label
                 # alpha near 0  -> pure teacher-matching
                 # alpha near 1  -> ignores teacher (plain training)

# sanity: at T=1 with alpha=1 this reduces to ordinary cross-entropy.
# scale the LR / schedule (Day 32) as usual; T^2 keeps soft grads comparable.

    Self-distillation & the "born-again" trick
    
        
    The student does not have to be smaller. Distilling a model into a same-size copy (or into itself across generations) often improves accuracy — the soft targets regularize better than one-hot labels. Chain a few generations and average.
    
        
    # "born-again networks": teacher and student share the architecture
teacher = model_gen0.eval()
for gen in range(1, 4):
    student = same_architecture()
    train(student, teacher, T=4.0, alpha=0.0)   # pure soft targets
    teacher = student.eval()                     # this gen teaches the next
# ensemble/average the generations for a final bump

    Where it shows up in the wild
    
        
    Distillation is how big models get shipped. DistilBERT keeps ~97% of BERT's performance at 40% the size and 60% faster. On-device vision/speech models are almost always distilled. And modern LLM pipelines distill reasoning traces: a large model generates step-by-step solutions, a small model is trained to reproduce them.
    
        
    # pattern: distill a large model's *outputs* (or reasoning) into a small one
big   = load("teacher-70B")           # slow, expensive to serve
small = load("student-7B")            # what you actually deploy

# 3. deploy `small`: ~same task quality, a fraction of the latency/cost

🎉 Day 33 of DeepLearningFromZero done.
Knowledge distillation trains a small student to imitate a big teacher, not by its final label but by its whole softened probability distribution. Dividing logits by a temperature T>1 flattens that distribution and reveals the dark knowledge — the teacher's sense of how the classes relate (a cat looks a little like a dog, nothing like a car) — information a one-hot label throws away entirely. The loss blends two teachers: cross-entropy to the true label and T²·KL to the teacher's soft targets, weighted by α; the T² factor rescales the soft gradients (which shrink as 1/T²) so both terms pull with comparable strength. The student often generalizes better than one trained on hard labels alone, because soft targets carry a richer, less overconfident signal. It is complementary to quantization and pruning (you can stack all three), and it is how DistilBERT, on-device models, and small reasoning models are made. 👉 Tomorrow, Day 34: VAEs — autoencoders that learn a smooth, samplable latent distribution via the reparameterization trick.
