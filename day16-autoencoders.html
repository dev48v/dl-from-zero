🔨 An autoencoder, in code

    
Define an encoder that shrinks the input to a tiny bottleneck → a decoder that rebuilds it → train with MSE reconstruction loss so the output matches the input → then use the latent code, or corrupt the input to train a denoiser. ~7 steps in PyTorch. Every line is the real thing — an autoencoder is just two small nets glued at a narrow waist.

    
    
      
  1. 1
    Pick the shapes — input size and bottleneck dim
    
        
    import torch, torch.nn as nn, torch.nn.functional as F

INPUT_DIM  = 28 * 28      # e.g. a flattened 28x28 image = 784 numbers
LATENT_DIM = 16           # the BOTTLENECK: squeeze 784 -> 16 numbers
# smaller LATENT_DIM = harder squeeze = the net must keep only essentials
# (this is the slider in the LOOK demo)
    
          
    2. 

      
  2. 2
    The encoder — funnel down to the bottleneck
    
        
    # the encoder maps the big input down to a tiny latent code z
encoder = nn.Sequential(
    nn.Linear(INPUT_DIM, 128), nn.ReLU(),
    nn.Linear(128, 64),        nn.ReLU(),
    nn.Linear(64, LATENT_DIM),                # <- the narrow waist (z)
)
# z = encoder(x)  is the COMPRESSED representation of the input
    
          
    3. 

      
  3. 3
    The decoder — fan back out to full size
    
        
    # the decoder is a mirror image: latent code -> rebuilt input
decoder = nn.Sequential(
    nn.Linear(LATENT_DIM, 64), nn.ReLU(),
    nn.Linear(64, 128),        nn.ReLU(),
    nn.Linear(128, INPUT_DIM), nn.Sigmoid(),  # back to 784, in [0,1]
)
# x_hat = decoder(z)  is the RECONSTRUCTION, same shape as the input
    
          
    4. 

      
  4. 4
    Glue them — forward pass = encode then decode
    
        
    class AutoEncoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.enc, self.dec = encoder, decoder
    def forward(self, x):
        z     = self.enc(x)      # compress  x -> z   (bottleneck)
        x_hat = self.dec(z)      # rebuild   z -> x_hat
        return x_hat, z

model = AutoEncoder()
    
          
    5. 

      
  5. 5
    Reconstruction loss — the target IS the input
    
        
    opt = torch.optim.Adam(model.parameters(), lr=1e-3)

def train_step(x):
    x_hat, z = model(x)
    loss = F.mse_loss(x_hat, x)        # output must match the INPUT
    opt.zero_grad(); loss.backward(); opt.step()
    return loss.item()
# no labels needed! the data is its own target -> self-supervised
    
          
    6. 

      
  6. 6
    Train — show it data, no labels
    
        
    for epoch in range(epochs):
    for x, _ in dataloader:            # ignore the label entirely
        x = x.view(x.size(0), -1)      # flatten to (batch, 784)
        loss = train_step(x)
    print(epoch, loss)
# because the bottleneck is narrow, the net CAN'T just copy x across;
# it is forced to learn the few features that best rebuild the data
    
          
    7. 

      
  7. 7
    Use it — latent, denoise, anomaly score
    
        
    # A) compressed representation (dim-reduction, clustering, search):
z = model.enc(x)                       # 784 -> 16 numbers

# B) DENOISING autoencoder: feed CORRUPTED input, target the CLEAN one
x_noisy = x + 0.3 * torch.randn_like(x)
x_clean, _ = model(x_noisy)            # bottleneck drops the noise
loss = F.mse_loss(x_clean, x)          # clean image is the target

# C) ANOMALY detection: high reconstruction error = doesn't fit the data
err = F.mse_loss(model(x)[0], x, reduction='none').mean(1)
is_anomaly = err > threshold           # weird inputs reconstruct badly

🎉 Day 16 of DeepLearningFromZero done.
Two small nets glued at a narrow waist: an encoder that compresses and a decoder that rebuilds, trained to make output ≈ input. The bottleneck forces it to keep only the essence — giving you a compressed latent code for dimensionality reduction, denoising, and anomaly detection. Make that latent probabilistic and you get a VAE that can generate.
