📐 Positional Encoding — attention reads every token at once, so order has to be injected by hand

A Transformer sees a bag of tokens, not a sequence. Position is something you add.

Self-attention is permutation-invariant — shuffle the tokens and the outputs just shuffle with them, so the model cannot tell token 1 from token 100. The original Transformer fixes this by adding a fixed positional-encoding vector to every token embedding. Below is the real sinusoidal encoding: PE(pos,2i)=sin(pos/100002i/d) and PE(pos,2i+1)=cos(pos/100002i/d). The left panel is the classic position × dimension heatmap — its signature vertical stripes come from a geometric ladder of frequencies: low dimensions oscillate fast, high dimensions barely move. Click any row to read that position's encoding vector and its per-dimension formula. The right panel is the payoff: the similarity between two positions depends only on their offset (it peaks at 0 and decays with distance), which is exactly why attention can learn relative position. 🔨 Build it yourself

    
You just saw that self-attention is permutation-blind, that a fixed sinusoidal vector added to each token fixes it, and that the dot product of two encodings depends only on their offset. Here it is in code: prove attention ignores order, build the sinusoidal table from the raw formula, add it to embeddings, verify the relative-offset property, then swap in a learned table and finally RoPE — the rotary scheme almost every modern LLM now uses.

    
    
      
  1. 
        
    1
    Prove it: attention is permutation-invariant
    
        
    Run plain self-attention on a sequence, then on a shuffled copy. The outputs come back in the same shuffled order, value-for-value — no positional information survives. That is the entire problem positional encoding exists to solve.
    
        
    import numpy as np
np.random.seed(0)

def attention(X):                      # X: (seq, d)  — no position info
    Q = K = V = X
    A = Q @ K.T / np.sqrt(X.shape[1])
    A = np.exp(A - A.max(1, keepdims=True)); A /= A.sum(1, keepdims=True)
    return A @ V

X = np.random.randn(4, 8)
perm = [2, 0, 3, 1]
out_a = attention(X)[perm]             # attend, then permute
out_b = attention(X[perm])             # permute, then attend
print(np.allclose(out_a, out_b))       # True -> attention is order-blind

    The sinusoidal table from the raw formula
    
        
    Even dimensions get a sine, odd dimensions a cosine, at a frequency ωᵢ = 10000−2i/d. As i grows the wavelength grows geometrically, from 2π up to ~10000·2π — the frequency ladder you see as stripes in the heatmap.
    
        
    import numpy as np

def sinusoidal_pe(seq_len, d):
    pos = np.arange(seq_len)[:, None]              # (seq, 1)
    i   = np.arange(d // 2)[None, :]               # (1, d/2)
    omega = 10000.0 ** (-(2 * i) / d)              # geometric frequency ladder
    pe = np.zeros((seq_len, d))
    pe[:, 0::2] = np.sin(pos * omega)              # even dims -> sin
    pe[:, 1::2] = np.cos(pos * omega)              # odd  dims -> cos
    return pe

PE = sinusoidal_pe(32, 16)
print(PE[0])          # position 0: sin=0, cos=1  -> [0,1,0,1,...]
# low dims (small i) oscillate fast; high dims (large i) barely move

    3
    Add it to the token embeddings
    
        
    Positional encoding is added (not concatenated) to the token embeddings before the first attention layer, so the width d stays the same and the network learns to blend content and position. Now two identical tokens at different positions have different inputs.
    
        
    import numpy as np

vocab, d, seq_len = 1000, 16, 32
token_embed = np.random.randn(vocab, d) * 0.1     # learned lookup table
PE = sinusoidal_pe(seq_len, d)                    # from step 2

tokens = np.array([5, 42, 42, 7])                 # note: token 42 twice
x = token_embed[tokens] + PE[:len(tokens)]        # add position in-place
# the two 42s are now DIFFERENT vectors: content is shared, position is not
print(np.allclose(x[1], x[2]))                    # False

    Verify the relative-offset property
    
        
    The dot product of two sinusoidal encodings is ∑ᵢ cos((pos−pos')·ωᵢ) — a function of the offset only. So the similarity between positions 3&5 equals that of 30&32. This is the similarity-vs-offset curve in the demo, and the reason attention can learn "look k tokens back".
    
        
    import numpy as np
PE = sinusoidal_pe(64, 16)

def sim(a, b):                                     # cosine similarity
    return PE[a] @ PE[b] / (np.linalg.norm(PE[a]) * np.linalg.norm(PE[b]))

print(round(sim(3, 5),  6))    # offset 2
print(round(sim(30, 32),6))    # offset 2 -> SAME value (depends on offset only)
print(round(sim(10, 10),6))    # offset 0 -> 1.0 (peak)
# PE(pos+k) is a fixed linear (rotation) function of PE(pos)

    Learned positional embeddings (BERT / GPT-2)
    
        
    Skip the formula and just make position a trainable lookup table. Simple and often slightly better in-distribution, but it has a hard cap at max_len and cannot extrapolate to longer sequences — the row for position 5000 was never trained.
    
        
    import torch, torch.nn as nn

class LearnedPE(nn.Module):
    def __init__(self, max_len, d):
        super().__init__()
        self.pos = nn.Embedding(max_len, d)        # trainable, one row per position
    def forward(self, x):                          # x: (batch, seq, d)
        idx = torch.arange(x.size(1), device=x.device)
        return x + self.pos(idx)                   # add, like sinusoidal

# tradeoff: adapts to the task, but hard-capped at max_len (no extrapolation)

    RoPE — rotate Q and K (the modern default)
    
        
    RoPE (LLaMA, GPT-NeoX, PaLM) adds nothing to the embeddings. It rotates each query and key by an angle pos·ωᵢ on every 2-D sub-pair, right before the dot product. Because a dot of two rotated vectors depends only on the difference of angles, the attention score becomes an explicit function of the relative offset — parameter-free and better at extrapolation.
    
        
    import numpy as np

def rope(x, pos):                        # x: (d,) with d even
    d = x.shape[0]
    i = np.arange(d // 2)
    theta = pos * (10000.0 ** (-(2 * i) / d))     # angle per 2-D pair
    x1, x2 = x[0::2], x[1::2]
    xr = np.empty_like(x)
    xr[0::2] = x1 * np.cos(theta) - x2 * np.sin(theta)   # rotate each pair
    xr[1::2] = x1 * np.sin(theta) + x2 * np.cos(theta)
    return xr

q = np.random.randn(8); k = np.random.randn(8)
# score(rope(q,m), rope(k,n)) depends on (m - n): relative position, for free
print(rope(q, 3) @ rope(k, 5))           # == rope(q,30) @ rope(k,32)

🎉 Day 29 of DeepLearningFromZero done.
Self-attention is permutation-invariant, so a Transformer has no built-in sense of order — you must inject it. The original fix adds a fixed sinusoidal vector to each token embedding: even dims are sin(pos/100002i/d), odd dims cos(·), forming a geometric frequency ladder (fast low dims, slow high dims — the heatmap stripes). Sinusoids are bounded, unique per position, and extend to any length, and their dot product depends only on the offset, so PE(pos+k) is a fixed linear function of PE(pos) — attention gets relative position for free. Learned tables adapt but cannot extrapolate past max_len; RoPE rotates Q/K by pos·ω and has become the default in modern LLMs. The arc runs from absolute added vectors toward relative offsets baked into the attention score. 👉 Tomorrow, Day 30: Multi-Head Attention — one attention is a single viewpoint; run h of them in parallel on different subspaces and concatenate, and the model reads several relationships at once.
