DeepLearningFromZero · Day 39

🕸️ DenseNet — ResNet adds a skip connection (y = x + F(x)), which can dilute what earlier layers computed. DenseNet goes to the other extreme: inside a dense block, every layer is wired to every layer before it — layer ℓ receives the concatenation of the feature-maps of all preceding layers and passes its own maps on to all that follow. Each layer adds only k new feature-maps (the growth rate, small — 12/24/32), so channels grow linearly k₀ + k·ℓ and the block holds L(L+1)/2 direct connections. Transition layers with a compression factor θ shrink channels between blocks. The payoff of all that feature reuse: state-of-the-art accuracy with a fraction of ResNet’s parameters

Wire every layer to every later layer. Drag the knobs — watch the channels stack up and the connections multiply.

Inside a dense block the input x₀ (with k₀ channels) flows down through L layers. Layer ℓ doesn’t just see the previous layer — it sees the concatenation of every feature-map produced so far, so its input has k₀ + k·(ℓ−1) channels. It then produces just k new maps (the growth rate) and hands them to all the layers below. Because every earlier map is still reachable, nothing has to be re-learned — that is feature reuse, and it’s why DenseNet is so parameter-thrifty. Drag k, L, k₀ and the compression θ and watch the channel math, the L(L+1)/2 connection count, and the ResNet contrast update live.

🔨 Build it yourself

    
The live demo is a from-scratch DenseNet skeleton — no library, just concatenation, a growth rate, and a compression factor. We assemble it one line at a time: the composite layer H that produces k new maps, the dense block that keeps concatenating them, the channel bookkeeping (k₀ + k·ℓ), the connection count, the transition layer that compresses by θ, the full network, the one-line diff against a ResNet block, and finally the couple of lines you actually write with torchvision.

    
    
      
  1. 
        
    1
    The composite layer H — each layer makes only k new maps
    
        
    One layer is H = BN → ReLU → 3×3 conv, and it outputs exactly k feature-maps — the growth rate. k is small (12, 24, 32) on purpose: a layer only has to contribute a thin slice of new information, because everything computed before it is still available by concatenation.
    
        
    // composite function H_l: BN -> ReLU -> 3x3 conv, outputs k maps
function H(x, k) {
  x = relu(batchnorm(x));
  return conv3x3(x, /*out_channels=*/ k);   // k = growth rate (small!)
}
// optional bottleneck (DenseNet-B): a 1x1 conv -> 4k maps first,
// to cap the cost of a very wide concatenated input.

  2. 
        
    2
    The dense block — concatenate, don’t add
    
        
    Keep a running list of every feature-map produced so far. To run layer ℓ, concatenate the whole list along the channel axis, feed it to H, and append the k new maps back onto the list. This is the one idea that defines DenseNet: concatenation, so each layer sees all prior features directly.
    
        
    function denseBlock(x0, L, k) {
  let feats = [x0];                       // collect all feature-maps
  for (let l = 1; l <= L; l++) {
    const inp = concat(feats);            // [x0, x1, ..., x_{l-1}]
    const xl  = H(inp, k);                // k new maps
    feats.push(xl);                       // reuse forever after
  }
  return concat(feats);                   // block output
}

    3
    Channel bookkeeping — growth is linear, not explosive
    
        
    Layer ℓ receives k₀ + k·(ℓ−1) channels and adds k. After all L layers the block output is k₀ + k·L. It feels like everything connects to everything, but the width only grows by a constant k per layer — linear, and cheap to keep in check.
    
        
    function inChannels(l, k0, k) { return k0 + k * (l - 1); }  // into layer l
function blockOutChannels(L, k0, k) { return k0 + k * L; }  // after L layers
// k0=24, k=12, L=6  ->  layer inputs: 24,36,48,60,72,84
//                       block output: 24 + 12*6 = 96 channels

  4. 
        
    4
    Counting connections — L(L+1)/2
    
        
    A plain L-layer chain has L connections (each layer feeds the next). In a dense block, layer ℓ is fed by ℓ sources (x₀ through xℓ₋₁), so the total is 1 + 2 + … + L = L(L+1)/2 direct connections — a quadratic web of short paths from any layer to any later one.
    
        
    function connections(L)      { return L * (L + 1) / 2; }  // dense
function plainConnections(L) { return L; }                 // a chain
// L = 6  ->  dense: 21 connections,  plain: 6
// every layer has a short path to every later layer -> strong gradients

  5. 
        
    5
    Transition layer — compress by θ between blocks
    
        
    You cannot concatenate across a downsample (feature-maps change size), and channels would balloon across many blocks. So between dense blocks sits a transition: a 1×1 conv that squeezes m channels down to ⌊θ·m⌋ (compression, DenseNet-C uses θ=0.5), then a 2×2 average-pool that halves the spatial size.
    
        
    function transition(x, m, theta) {          // m = incoming channels
  x = conv1x1(x, Math.floor(theta * m));    // compress channels (theta<=1)
  x = avgPool2x2(x);                        // halve H and W
  return x;                                 // now smaller & thinner
}
// theta = 1.0 -> no compression ; theta = 0.5 -> DenseNet-C / -BC

    Assemble the network — stem, blocks, transitions, head
    
        
    A full DenseNet is: a conv stem, then several dense blocks separated by transition layers, then a global average pool and a linear classifier. DenseNet-121 uses four blocks of [6, 12, 24, 16] layers with k = 32.
    
        
    function DenseNet(x, blocks, k, k0, theta) {
  let ch = stemConv(x, k0);                 // -> k0 channels
  blocks.forEach((L, i) => {
    ch = denseBlock(ch, L, k);              // width += k each layer
    let m = channelsOf(ch);
    if (i < blocks.length - 1)              // transition after all but last
      ch = transition(ch, m, theta);        // compress + downsample
  });
  return linear(globalAvgPool(ch), numClasses);
}
// DenseNet-121: blocks=[6,12,24,16], k=32, theta=0.5

    7
    The whole difference from ResNet, in one line
    
        
    ResNet and DenseNet both give early layers a short path to the loss. ResNet does it by summing a same-shaped residual back on; DenseNet does it by concatenating new maps and keeping the old ones. Summing can overwrite; concatenating preserves — which is why DenseNet reuses features so aggressively.
    
        
    // ResNet block:   add — width stays the same, features get summed
y = x.add( F(x) );                 //  y = x + F(x)

// DenseNet layer:  concat — width grows by k, every map kept
y = concat([ x, F(x) ]);           //  y = [x, F(x)]  (F outputs k maps)
// add can dilute earlier signal; concat guarantees it survives intact.

    The real thing — a few lines in torchvision
    
        
    You never hand-roll the concatenation. Torchvision ships the whole family — densenet121/169/201/161 — already wired with dense blocks, bottlenecks and θ=0.5 transitions. DenseNet-121 matches a much heavier ResNet on ImageNet with roughly a third of the parameters, purely from feature reuse.
    
        
    import torchvision.models as models

net = models.densenet121(weights="IMAGENET1K_V1")  # k=32, blocks=[6,12,24,16]
# ~8M params vs ResNet-50's ~25M — similar ImageNet accuracy.

# build-your-own knobs (torchvision.models.DenseNet):
net = models.DenseNet(growth_rate=32,
                      block_config=(6, 12, 24, 16),
                      num_init_features=64,       # k0 (stem)
                      bn_size=4,                   # bottleneck: 1x1 -> 4k
                      compression=0.5)             # theta at transitions
# lineage of feature-reuse nets: DenseNet -> DPN -> CSPNet (used in YOLO).

🎉 Day 39 of DeepLearningFromZero done.
DenseNet takes ResNet’s short-path idea to its logical extreme: instead of adding a residual (y = x + F(x)), every layer inside a dense block receives the concatenation of the feature-maps of all preceding layers and passes its own maps to all that follow — giving L(L+1)/2 direct connections. Each layer contributes only k new feature-maps (the small growth rate), so channels grow just linearly as k₀ + k·ℓ; transition layers with a compression factor θ squeeze channels and downsample between blocks. Because every earlier feature stays directly reachable, nothing has to be re-learned — that feature reuse gives every layer a short path to the loss (strong gradients, no vanishing) and buys top-tier accuracy at a fraction of ResNet’s parameter count (DenseNet-121 ≈ a ResNet-50 with ~3× fewer params). 👉 Tomorrow, Day 40: Squeeze-and-Excitation networks — instead of adding connections, let the network learn which channels matter and recalibrate them (channel-wise attention).
