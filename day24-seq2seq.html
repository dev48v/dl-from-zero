🔨 Build it yourself

    
You just watched a from-scratch encoder–decoder run in JS. Here's the same architecture in PyTorch — two GRUs, a context hand-off, and an autoregressive decode loop — the skeleton every seq2seq model (translation, summarisation) is built on.

    
    
      
  1. 
        
    1
    Define special tokens & vocab
    
        
    Every seq2seq model needs a <SOS> (start-of-sequence) to kick off decoding and an <EOS> (end-of-sequence) so the model can decide when to stop — outputs are variable length. Pad token fills out batches. These live in the vocabulary alongside the real symbols.
    
        
    PAD, SOS, EOS = 0, 1, 2
# toy task: reverse a digit string, e.g. [3,1,4] -> [4,1,3]
# vocab = {PAD, SOS, EOS, '0'..'9'}  -> 13 symbols
VOCAB = 13
    
      
    2. 
      
  2. 
        
    2
    The Encoder: sequence → context vector
    
        
    The encoder is an RNN (here a GRU) that reads the input one token at a time. We throw away its per-step outputs and keep only the final hidden state — that single fixed-size vector is the "context" (aka thought vector) meant to summarise the entire input.
    
        
    import torch, torch.nn as nn

class Encoder(nn.Module):
    def __init__(self, vocab, emb=32, hid=64):
        super().__init__()
        self.embed = nn.Embedding(vocab, emb)
        self.rnn   = nn.GRU(emb, hid, batch_first=True)
    def forward(self, x):                # x: (B, T_in)
        e = self.embed(x)
        outputs, h = self.rnn(e)         # h: (1, B, hid)
        return h                         # <- the context vector

  3. 
        
    3
    The Decoder: context → output sequence
    
        
    The decoder is a second GRU initialised with the encoder's context. At each step it takes the previous token, updates its hidden state, and projects to a distribution over the vocabulary. One step = predict one output token.
    
        
    class Decoder(nn.Module):
    def __init__(self, vocab, emb=32, hid=64):
        super().__init__()
        self.embed = nn.Embedding(vocab, emb)
        self.rnn   = nn.GRU(emb, hid, batch_first=True)
        self.out   = nn.Linear(hid, vocab)
    def forward(self, tok, h):           # one step
        e = self.embed(tok)              # (B, 1, emb)
        o, h = self.rnn(e, h)            # h carries state forward
        logits = self.out(o.squeeze(1))  # (B, vocab)
        return logits, h

    Training with teacher forcing
    
        
    During training we feed the true previous target token (not the model's own guess) into each decoder step. This "teacher forcing" stabilises early learning and lets us compute the loss over the whole sequence at once with cross-entropy.
    
        
    enc, dec = Encoder(VOCAB), Decoder(VOCAB)
opt = torch.optim.Adam([*enc.parameters(), *dec.parameters()], 1e-3)
loss_fn = nn.CrossEntropyLoss(ignore_index=PAD)

for src, tgt in loader:               # tgt starts with SOS, ends EOS
    h = enc(src)                      # context
    loss = 0
    tok = tgt[:, :1]                  # 
    for t in range(1, tgt.size(1)):
        logits, h = dec(tok, h)
        loss += loss_fn(logits, tgt[:, t])
        tok = tgt[:, t:t+1]           # feed the TRUE token (teacher forcing)
    opt.zero_grad(); loss.backward(); opt.step()

    Inference: autoregressive decode until <EOS>
    
        
    At test time there are no true tokens — the decoder feeds its own prediction back in (free-running) and loops until it emits <EOS> or hits a max length. This is exactly the loop the LOOK demo animates.
    
        
    @torch.no_grad()
def translate(src, max_len=20):
    h = enc(src)
    tok = torch.tensor([[SOS]])
    out = []
    for _ in range(max_len):
        logits, h = dec(tok, h)
        tok = logits.argmax(-1, keepdim=True)  # feed own guess back
        nxt = tok.item()
        if nxt == EOS: break
        out.append(nxt)
    return out

    Feel the bottleneck → reach for attention
    
        
    Push the input length up and accuracy collapses: a single fixed vector can't hold a long sentence. The fix is attention — let the decoder look back at all encoder states, not just the last one. That idea, then dropping recurrence entirely, gave us the Transformer.
    
        
    # plain seq2seq: decoder sees only the last state h
logits, h = dec(tok, h)

# + attention (Day 12): decoder sees ALL encoder outputs
outputs, h = enc.rnn(enc.embed(src))     # keep every step
ctx = attention(query=h, keys=outputs, values=outputs)
logits, h = dec(tok, h, context=ctx)     # no bottleneck
# drop recurrence altogether -> Transformer (Day 13)
