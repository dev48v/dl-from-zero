🚪 GRU — two gates, one hidden state: the streamlined LSTM

One recurrent cell, two learned valves — and a hidden state that can hold a memory for as long as it needs.

A vanilla RNN (Day 10) crushes its hidden state through a tanh every step, so early signals vanish. The LSTM (Day 11) fixed that with three gates and a separate cell state. The GRU gets the same benefit with two gates and no extra state. This runs a real GRU cell on seeded weights — genuine sigmoid gates and a tanh candidate — over a tiny remember-the-first-token task. Watch the update gate z and reset gate r light up (0→1 heat bars) and the hidden state h update each timestep by the convex mix h = (1−z)·h_prev + z·h̃. Feed it a token, then a run of blanks: when the gate learns to stay shut, the first token is carried unchanged to the end; push the sliders and you can make it forget. All gate math is real.

🔨 Build it yourself

    
You just watched a real GRU cell carry a token across a run of blanks using nothing but its update gate. Here it is in PyTorch: the one-line nn.GRU, the exact cell equations spelled out in a GRUCell, the shapes you actually pass, a sequence-classifier head for the remember-first-token task, and the one-line swap to an LSTM so you can benchmark both.

    The whole layer is one line: nn.GRU
    
        
    PyTorch ships the GRU as a fused, cuDNN-accelerated layer. You give it the input feature size and the hidden size; it returns the full sequence of hidden states plus the final one. batch_first=True makes the tensors read (batch, time, features).
    
        
    import torch, torch.nn as nn

gru = nn.GRU(input_size=16, hidden_size=32,
             num_layers=1, batch_first=True)

x = torch.randn(8, 5, 16)          # (batch=8, seq_len=5, features=16)
out, h_n = gru(x)                  # out:(8,5,32)  h_n:(1,8,32)
# out[:, t, :] = hidden state after timestep t ; h_n = out[:, -1, :]

    The four equations, written out as a GRUCell
    
        
    This is exactly what the LOOK demo computes at every timestep: two sigmoid gates, one tanh candidate, and the convex-combination update. Follow it once and the fused layer is no longer a black box.
    
        
    import torch, torch.nn as nn, torch.nn.functional as F

class GRUCell(nn.Module):
    def __init__(self, d_in, d_h):
        super().__init__()
        self.x2h = nn.Linear(d_in, 3 * d_h)   # W_z, W_r, W_h stacked
        self.h2h = nn.Linear(d_h,  3 * d_h)   # U_z, U_r, U_h stacked

    def forward(self, x, h_prev):
        xz, xr, xh = self.x2h(x).chunk(3, dim=-1)
        hz, hr, hh = self.h2h(h_prev).chunk(3, dim=-1)
        z = torch.sigmoid(xz + hz)            # update gate  in [0,1]
        r = torch.sigmoid(xr + hr)            # reset  gate  in [0,1]
        h_tilde = torch.tanh(xh + r * hh)     # candidate (reset filters the past)
        return (1 - z) * h_prev + z * h_tilde # h = (1-z)*h_prev + z*h~

    Roll the cell over a sequence
    
        
    The fused nn.GRU does this loop in C for speed, but writing it once shows how the same weights are reused at every timestep and how the hidden state threads through — carrying information forward when z stays near zero.
    
        
    cell = GRUCell(d_in=16, d_h=32)

def run(seq):                          # seq: (batch, time, d_in)
    B, T, _ = seq.shape
    h = torch.zeros(B, 32)             # h_0
    outs = []
    for t in range(T):
        h = cell(seq[:, t, :], h)      # same weights every step
        outs.append(h)
    return torch.stack(outs, dim=1), h # (B,T,32), final h

    A classifier head for remember-first-token
    
        
    Take the final hidden state and map it to a label. Train this on sequences whose answer depends only on the first token and the GRU learns, on its own, to shut the update gate on the blank steps so the first token survives — the behaviour the slider forces by hand in LOOK.
    
        
    class FirstTokenClassifier(nn.Module):
    def __init__(self, vocab, d_emb=16, d_h=32, n_cls=3):
        super().__init__()
        self.emb = nn.Embedding(vocab, d_emb)
        self.gru = nn.GRU(d_emb, d_h, batch_first=True)
        self.fc  = nn.Linear(d_h, n_cls)
    def forward(self, tokens):          # tokens: (batch, time) of ids
        _, h_n = self.gru(self.emb(tokens))
        return self.fc(h_n[-1])         # logits from the FINAL hidden state

model = FirstTokenClassifier(vocab=4)
loss  = F.cross_entropy(model(tokens), labels)   # standard training loop

    Stack, go bidirectional, add dropout
    
        
    The same knobs as nn.LSTM. Depth stacks GRUs; bidirectional runs a second GRU backwards and concatenates, doubling the output width; inter-layer dropout regularises. Remember to sort/pack variable-length batches.
    
        
    gru = nn.GRU(input_size=16, hidden_size=32,
             num_layers=2,        # stacked depth
             bidirectional=True,  # out width = 2 * hidden_size
             dropout=0.3,         # applied between layers
             batch_first=True)

# variable-length batches: pack so padding is skipped
from torch.nn.utils.rnn import pack_padded_sequence
packed = pack_padded_sequence(x, lengths, batch_first=True,
                              enforce_sorted=False)
out, h_n = gru(packed)

    GRU or LSTM? Swap one line and measure
    
        
    The interfaces are identical except the LSTM's hidden state is a (h, c) tuple. There is no universal winner — GRU is lighter and often trains faster on small/medium data; LSTM sometimes edges ahead on very long dependencies. Benchmark both on your task.
    
        
    # identical call site — just change the module:
rnn = nn.GRU (d_in, d_h, batch_first=True)   # 3 weight blocks, state = h
rnn = nn.LSTM(d_in, d_h, batch_first=True)   # 4 weight blocks, state = (h, c)

# rule of thumb
small_or_fast_data   -> GRU     # fewer params, quicker to train
very_long_context    -> LSTM    # extra cell state can help
production_2024_plus -> Transformer  # attention replaced RNNs for most seq tasks

🎉 Day 26 of DeepLearningFromZero done.
You ran a real GRU cell: two sigmoid gates and a tanh candidate feeding the convex update h = (1−z)·h_prev + z·h̃. You saw the update gate z act as a write/carry valve — near 0 it copies the old state forward untouched (the additive-style path that lets gradients survive, the fix for the vanilla RNN's vanishing gradient), near 1 it refreshes to the candidate — and the reset gate r decide how much of the past even enters that candidate. Two gates and one state give the GRU most of the LSTM's long-range memory with fewer parameters and no separate cell state. 👉 Tomorrow — Day 27: Attention recap / Layer Norm — the mechanism (and the normalisation trick) that let Transformers retire recurrence for most sequence tasks.
