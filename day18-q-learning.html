↑ The agent isn't told the right moves — it tries actions, gets rewards, and updates a Q-table so that, over many episodes, it learns which action in each state leads to the most future reward. This is tabular Q-learning: it stores one number per (state, action). Real problems have too many states for a table, so Deep Q-Networks (DQN) replace the table with a neural network that predicts Q-values — that's deep reinforcement learning.

🔨 Q-learning, in code

    
Define the environment (states, actions, rewards) → make an empty Q-table → pick actions ε-greedily (mostly best, sometimes random) → take a step, see the reward, and apply the Bellman update → loop over many episodes → read off the greedy policy the table now encodes. ~7 steps of plain Python — the whole learning rule is a single line. At the end: the same loop with Gymnasium environments, and the one change that turns it into DQN.

    
    
      
  1. 1
    Define the environment: states, actions, rewards
    
        
    import numpy as np, random

GRID = 5                       # 5x5 gridworld -> 25 states
ACTIONS = [0, 1, 2, 3]         # 0=up 1=down 2=left 3=right
GOAL, PIT = 24, 12             # cell indices: +1 goal, -1 pit
START = 0

def step(state, a):
    r, c = divmod(state, GRID)
    if a == 0: r -= 1
    elif a == 1: r += 1
    elif a == 2: c -= 1
    elif a == 3: c += 1
    if not (0 <= r < GRID and 0 <= c < GRID):   # wall: stay put
        r, c = divmod(state, GRID)
    ns = r * GRID + c
    if ns == GOAL: return ns,  1.0, True        # reward, done
    if ns == PIT:  return ns, -1.0, True
    return ns, -0.04, False                      # small step cost
    
          
    2. 

      
  2. 2
    Initialise the Q-table to all zeros
    
        
    # Q[state, action] = expected future reward of taking that action there.
# We know NOTHING yet, so start every value at 0.
Q = np.zeros((GRID * GRID, len(ACTIONS)))     # shape (25, 4)

alpha = 0.5     # learning rate: how much each update moves Q
gamma = 0.9     # discount: how much we value future vs immediate reward
epsilon = 0.3   # exploration: chance of trying a random action
    
          
    3. 

      
  3. 3
    Pick an action ε-greedily (explore vs exploit)
    
        
    def choose(state):
    if random.random() < epsilon:
        return random.choice(ACTIONS)      # EXPLORE: try something random
    return int(np.argmax(Q[state]))        # EXPLOIT: take the best known action

# early on, exploration discovers the goal/pits;
# later, exploitation follows the best path found so far.
    
          
    4. 

      
  4. 4
    Take a step and apply the Bellman update
    
        
    # this single line IS Q-learning:
def update(s, a, r, ns, done):
    best_next = 0 if done else np.max(Q[ns])          # best future from next state
    target = r + gamma * best_next                    # reward + discounted future
    Q[s, a] += alpha * (target - Q[s, a])             # nudge Q toward the target
# target - Q[s,a] is the "TD error": how wrong our old estimate was.
# alpha controls how big a correction we make each time.
    
          
    5. 

      
  5. 5
    Loop over many episodes
    
        
    for episode in range(2000):
    s, done = START, False
    while not done:
        a = choose(s)                # ε-greedy action
        ns, r, done = step(s, a)     # environment responds
        update(s, a, r, ns, done)    # learn from it
        s = ns
    epsilon = max(0.02, epsilon * 0.999)   # explore less over time
# each episode = one attempt from start to goal/pit.
# values propagate backward from the goal a little more every episode.
    
          
    6. 

      
  6. 6
    Extract the greedy policy the table learned
    
        
    # the policy is just "argmax over actions" in each state.
policy = np.argmax(Q, axis=1)        # best action per state
names = {0:"↑", 1:"↓", 2:"←", 3:"→"}

# walk it greedily — no exploration — to follow the optimal path:
s, path = START, [START]
while s not in (GOAL, PIT):
    s, _, done = step(s, int(policy[s]))
    path.append(s)
    if done: break
print("learned route:", path)

s, _ = env.reset()

# --- the deep-RL leap: replace the TABLE with a NEURAL NET ---
# A table needs one row per state -> impossible for images / huge spaces.
# DQN trains a network Q(s) ~= row of Q-values, learned by the SAME
# Bellman target, plus a replay buffer + a slow "target network":
#   loss = ( r + gamma * max_a' Q_target(s', a')  -  Q(s, a) )**2
# This is how agents learned to play Atari straight from pixels.

🎉 Day 18 of DeepLearningFromZero done.
Reinforcement learning learns from rewards, not labels. The agent loops through state → action → reward → next state, and one Bellman update — Q(s,a) += α·(r + γ·maxQ(s',·) − Q(s,a)) — slowly fills a Q-table with the expected future reward of every move. ε-greedy balances trying new things against using what works; γ decides how far ahead it plans. Tabular Q-learning can't fit problems with millions of states, so DQN swaps the table for a neural network that predicts Q-values — and that's deep reinforcement learning, the foundation behind game-playing and robotics agents.
