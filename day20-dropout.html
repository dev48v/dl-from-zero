🎲 Dropout — Switch Off Neurons to Generalize

Watch neurons randomly switch off each step — and the overfit gap shrink

A small fully-connected net runs left→right. During TRAINING with dropout, each forward pass randomly drops a fraction p of the hidden neurons — they gray out and their edges are cut, and a different random subset goes dark every step. The net can't lean on any single neuron, so it learns redundant, robust features instead of memorizing — like training a whole ensemble of subnetworks. The two loss-style curves show the payoff: without dropout the train accuracy races to ~100% but the validation curve overfits and the gap grows; with dropout the gap stays small (better generalization). At INFERENCE every neuron is on (no dropping), with activations scaled to compensate. Slide dropout p and press Train.

↑ Dropout randomly switches off neurons during training so the network learns redundant, robust features instead of memorizing — like training an ensemble of many subnetworks that share weights. Because no neuron can count on its neighbours always being present, it breaks co-adaptation and acts as regularization, shrinking the overfit gap. At test time everything's on: no neuron is dropped, and the activations are scaled (inverted dropout multiplies by 1/(1−p) during training) so the expected signal matches and inference needs no change.

🔨 Dropout, in code

    
Take a layer's activations → during training sample a random keep mask (each unit kept with prob 1−p) → zero out the dropped units → scale up the survivors by 1/(1−p) so the expected output is unchanged (inverted dropout) → at eval time do nothing (all units on, no scaling) → and in real code just drop in nn.Dropout(p) or Keras Dropout(p). ~7 steps of plain Python — the whole transform is three short lines.

    
    
      
  1. 1
    Start from a layer's activations
    
        
    import numpy as np

# a is the output (activations) of a hidden layer for a mini-batch:
#   shape (batch_size, num_units)
batch_size, num_units = 32, 64
a = np.random.rand(batch_size, num_units)   # some positive activations

p = 0.5          # DROP probability (fraction to switch off); keep = 1 - p
keep = 1 - p     # probability a unit survives
    
          
    2. 

      
  2. 2
    Sample a random keep mask (training only)
    
        
    # draw one Bernoulli per unit: 1 = keep, 0 = drop.
# a fresh, different mask is drawn on EVERY forward pass.
mask = (np.random.rand(*a.shape) > p).astype(a.dtype)

# roughly a fraction p of the entries are 0 (those units are switched off).
print("kept:", mask.mean().round(2), " (~", round(keep,2), ")")
    
          
    3. 

      
  3. 3
    Zero out the dropped units
    
        
    # multiply elementwise: dropped units become 0, survivors pass through.
a_drop = a * mask

# the network for THIS step is a random subnetwork: only the kept units
# (and the edges into/out of them) participate in forward & backward.
    
          
    4. 

      
  4. 4
    Scale the survivors by 1/(1−p) — "inverted dropout"
    
        
    # zeroing units lowers the layer's expected sum. To keep the expected
# output the SAME as without dropout, scale the survivors back up:
a_drop = a_drop / keep        # == a * mask / (1 - p)

# doing the scaling at TRAIN time is "inverted dropout": it means
# inference can run untouched (no scaling needed there).
    
          
    5. 

      
  5. 5
    At eval time: do nothing (all units on)
    
        
    # inference uses the WHOLE network, no dropping and no scaling,
# because inverted dropout already corrected the scale during training.
def dropout(a, p, training):
    if not training or p == 0:
        return a                     # eval: pass activations through unchanged
    mask = (np.random.rand(*a.shape) > p) / (1 - p)
    return a * mask                  # train: drop + scale survivors
    
          
    6. 

      
  6. 6
    Where to put it & which rate to use
    
        
    # put dropout AFTER the activation, on dense (fully-connected) layers,
# which have the most parameters and overfit most.
# typical p: 0.2-0.5 (higher near big dense layers, lower/none in conv).
hidden = relu(linear(x))
hidden = dropout(hidden, p=0.5, training=True)   # heavy on dense layers
out    = linear(hidden)
# convolutions share weights & overfit less -> use little or no dropout.
    
          
    7. 

      
  7. 7
    In real code: nn.Dropout & Keras Dropout
    
        
    import torch.nn as nn

# --- PyTorch does all of the above for you (inverted dropout built in) ---
net = nn.Sequential(
    nn.Linear(64, 128), nn.ReLU(),
    nn.Dropout(0.5),                 # drop 50% of these activations in training
    nn.Linear(128, 10),
)
# model.train() -> dropout ACTIVE   |   model.eval() -> dropout OFF (all units)

# --- Keras ---
# tf.keras.layers.Dropout(0.5)      # same idea; off automatically at predict()
# Transformers still use dropout in attention & the feed-forward (FFN) blocks.

🎉 Day 20 of DeepLearningFromZero done.
Dropout randomly zeros a fraction p of a layer's activations every training step — a fresh random subnetwork each pass. Because no unit can rely on any specific neighbour, the network is forced to learn redundant, robust features instead of memorizing, which breaks co-adaptation and works like training an ensemble of subnetworks that share weights. Survivors are scaled by 1/(1−p) (inverted dropout) so that at test time everything is on with no change needed. It's a regularizer like L2 (Day 17) and partners with batch norm (Day 19); with strong BN and large datasets it's needed less in conv nets, but it's still standard in Transformers' attention and feed-forward blocks.
