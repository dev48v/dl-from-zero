📏 Layer Normalization — normalize across features, per sample: the norm that powers Transformers

Normalization is just: subtract the mean, divide by the standard deviation.

Back on Day 19, BatchNorm normalized each feature down a column across every sample in the mini-batch. LayerNorm turns the grid 90 degrees: it normalizes each sample across its own features, along a row. That single change makes the statistics depend only on the one sample in front of you, never on the rest of the batch. Below is a real normalizer running on a fixed 4x6 batch (4 samples, 6 features): pick the axis, press Normalize, and watch every row (or column) snap to mean 0 / variance 1, then push the learnable gamma (scale) and beta (shift) sliders. Then drop the batch size to 1 and watch BatchNorm collapse while LayerNorm keeps working, the exact reason Transformers, RNNs and every modern LLM normalize per token, not per batch. All mean/variance math is genuine.

🔨 Build it yourself

    
You just watched a real normalizer standardize each row of a batch to mean 0 / variance 1, then rescale with learnable γ and β, and saw why doing it per sample (LayerNorm) instead of per feature-across-the-batch (BatchNorm) is what makes it batch-size independent. Here it is in PyTorch: the one-line nn.LayerNorm, the exact math spelled out, how it plugs into a Transformer token tensor, the pre-norm vs post-norm choice, the concrete differences from nn.BatchNorm1d, and the one-parameter RMSNorm that most 2023+ LLMs use instead.

    The whole layer is one line: nn.LayerNorm
    
        
    You give it normalized_shape, the trailing feature dimension(s) to normalize over. It computes the mean and variance across those features, for each sample independently, then applies a learnable per-feature gamma (scale) and beta (shift).
    
        
    import torch, torch.nn as nn

ln = nn.LayerNorm(normalized_shape=512)   # d_model = 512

x = torch.randn(8, 512)     # (batch=8, features=512)
y = ln(x)                   # each of the 8 rows -> mean 0, var 1, then gamma/beta
# y.shape == (8, 512) ; stats are computed PER ROW, never across the batch

    The math, written out
    
        
    This is exactly what the LOOK demo computes for every row: mean and variance over the last dimension, standardize, then the affine map. Note unbiased=False: LayerNorm uses the biased (population) variance, dividing by D, not D−1.
    
        
    import torch

def layer_norm(x, gamma, beta, eps=1e-5):
    # x: (..., D) -- normalize over the LAST dim (the features)
    mu   = x.mean(dim=-1, keepdim=True)                 # per-sample mean
    var  = x.var(dim=-1, unbiased=False, keepdim=True)  # per-sample variance
    xhat = (x - mu) / torch.sqrt(var + eps)             # -> mean 0, var 1
    return gamma * xhat + beta                          # learnable scale + shift

# gamma, beta are Parameters of shape (D,), initialised to 1 and 0

    Plug it into a Transformer: normalize each token
    
        
    A Transformer activation is (batch, seq_len, d_model). LayerNorm(d_model) normalizes each token over its own d_model features, never across other tokens or other sentences. That is why batch size and sequence length simply do not matter.
    
        
    # In a Transformer, x is (batch, seq_len, d_model).
# LayerNorm(d_model) normalizes EACH TOKEN over its d_model features,
# independently of the other tokens and of the batch.
x  = torch.randn(32, 100, 512)   # 32 sentences, 100 tokens, d_model=512
ln = nn.LayerNorm(512)
y  = ln(x)                       # (32,100,512): 3200 tokens each normalized alone
# batch=1, seq_len=7, or seq_len=4000 -- all identical maths, no cross-token stats

    Pre-norm vs post-norm placement
    
        
    The original 2017 Transformer put LayerNorm after the residual add (post-norm), which needs a learning-rate warmup to train deep stacks. Modern models (GPT-2 onward) put it before the sublayer (pre-norm), leaving the residual path a clean identity highway: far more stable and the default today.
    
        
    # Post-LN (original 2017 Transformer): norm AFTER the residual add
x = ln(x + sublayer(x))       # powerful but harder to train deep; needs warmup

# Pre-LN (GPT-2 onward): norm BEFORE the sublayer; add stays a clean highway
x = x + sublayer(ln(x))       # stable gradients, trains very deep nets easily

    LayerNorm vs BatchNorm, concretely
    
        
    BatchNorm keeps a running mean/variance and behaves differently in .train() vs .eval(); it also needs a reasonably sized batch. LayerNorm has no running stats and no train/eval switch: the same maths at any batch size, which is exactly why sequence models prefer it.
    
        
    bn = nn.BatchNorm1d(512)    # normalizes each FEATURE across the batch
ln = nn.LayerNorm(512)      # normalizes each SAMPLE across its features

bn.train(); bn(x)           # uses batch stats AND updates running mean/var
bn.eval();  bn(x)           # switches to the stored running averages
# LayerNorm has NO running stats and NO train/eval difference:
ln.eval();  ln(x)           # identical maths in train and eval, any batch size

    RMSNorm: the modern shortcut
    
        
    RMSNorm drops the mean-centering and the beta shift entirely: it just divides by the root-mean-square of the features and rescales. Fewer operations, no mean to compute, and in practice it works as well, so LLaMA, Mistral and most 2023+ LLMs use it in place of LayerNorm.
    
        
    class RMSNorm(nn.Module):
    def __init__(self, d, eps=1e-6):
        super().__init__()
        self.eps = eps
        self.g = nn.Parameter(torch.ones(d))   # scale only -- no beta, no mean
    def forward(self, x):
        rms = x.pow(2).mean(dim=-1, keepdim=True).add(self.eps).sqrt()
        return self.g * x / rms                # divide by RMS, skip centering
# used in LLaMA, Mistral, most 2023+ LLMs -- cheaper, matches LayerNorm quality

🎉 Day 27 of DeepLearningFromZero done.
You saw that normalization is one recipe, subtract the mean, divide by the std, then rescale with γ and β, and that the only thing separating BatchNorm from LayerNorm is which numbers the mean and variance are taken over. BatchNorm averages each feature down the batch, so its statistics depend on the other samples, differ between train and eval, and collapse when the batch is tiny. LayerNorm averages each sample across its own features, so it is completely batch-size independent, identical at train and inference, and normalizes every token on its own: the properties that let it slot into Transformers, RNNs and variable-length sequences where BatchNorm cannot. 👉 Tomorrow, Day 28: Word2Vec is already live back on Day 23, so next up is most likely RMSNorm (the centering-free norm you met in step 6) or Weight Initialization (Xavier / He), the other half of keeping activations healthy.
