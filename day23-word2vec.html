🧭 Word2Vec (skip-gram) — king − man + woman ≈ queen

Words have no meaning to a computer — until you train vectors for them.

This trains a real skip-gram model with negative sampling — right here in your browser, no libraries. It reads a tiny toy corpus about royalty, animals and food, and for every (center word, context word) pair it nudges their vectors together and pushes a few random "negative" words apart, using actual gradient updates. Watch the loss fall, then see the learned 12-dimensional vectors projected to 2D: related words cluster. Finally the analogy box computes king − man + woman and finds the nearest word — aiming for queen.

🔨 Build it yourself

    
You just watched a from-scratch skip-gram train in JS. In practice you'd reach for a battle-tested library. Here's the whole path — from raw text to trained embeddings to the analogy — in a few lines of gensim.

    
    
      
  1. 
        
    1
    Prepare a corpus of tokenised sentences
    
        
    Word2Vec learns from co-occurrence, so it just needs a list of sentences, each a list of lowercased tokens. Real training uses millions of sentences (Wikipedia, news dumps); the more text, the sharper the vectors. No labels — it's fully self-supervised from raw text.
    
        
    sentences = [
    "the king rules the kingdom".split(),
    "the queen rules the kingdom".split(),
    "the man is a king".split(),
    "the woman is a queen".split(),
    # ... thousands more, ideally millions
]
    
      
    2. 
      
  2. 
        
    2
    Train Word2Vec (skip-gram + negative sampling)
    
        
    sg=1 selects skip-gram (predict context from center); negative=5 turns on negative sampling instead of the full softmax; vector_size is the embedding dimension and window the context radius. This is exactly the algorithm the LOOK demo runs, just in optimised C.
    
        
    from gensim.models import Word2Vec

model = Word2Vec(
    sentences,
    vector_size=100,   # embedding dimension
    window=5,          # context radius
    sg=1,              # 1 = skip-gram, 0 = CBOW
    negative=5,        # negative samples per positive
    min_count=1,
    epochs=50,
)
    
      
    3. 
      
  3. 
        
    3
    Inspect vectors & nearest neighbours
    
        
    Every word now has a dense vector. most_similar ranks the vocabulary by cosine similarity — the same operation the demo does when you click a word. Semantically related words come out on top, even though you never told the model they were related.
    
        
    vec = model.wv["king"]          # a 100-d numpy array
print(vec.shape)                # (100,)

print(model.wv.most_similar("king", topn=5))
# [('queen', 0.78), ('prince', 0.71), ('kingdom', 0.66), ...]

print(model.wv.similarity("dog", "cat"))   # cosine, ~0.8
    
      
    4. 
      
  4. 
        
    4
    The analogy: king − man + woman ≈ queen
    
        
    Because gender is encoded as a roughly constant direction in vector space, subtracting "man" and adding "woman" shifts "king" along that direction and lands near "queen". most_similar(positive=..., negative=...) does the vector arithmetic and returns the closest word.
    
        
    print(model.wv.most_similar(
    positive=["king", "woman"],
    negative=["man"],
    topn=1,
))
# [('queen', 0.71)]   ← the famous analogy

# same trick works for others:
# paris - france + italy ≈ rome
    
      
    5. 
      
  5. 
        
    5
    Save, reload, or grab pretrained vectors
    
        
    Training is a one-time cost — persist the vectors and load them anywhere. Or skip training entirely and download Google's original 300-d Word2Vec (trained on 100B words of news) via gensim's downloader. These static vectors were the standard NLP input feature for years, until contextual models (ELMo, BERT) took over.
    
        
    model.wv.save_word2vec_format("vecs.txt")   # save
# ...later
from gensim.models import KeyedVectors
wv = KeyedVectors.load_word2vec_format("vecs.txt")

# or pretrained, no training needed:
import gensim.downloader as api
wv = api.load("word2vec-google-news-300")
