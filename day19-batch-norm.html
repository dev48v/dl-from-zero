📊 Batch Normalization — Keep Activations Healthy

Watch activations drift and saturate — then snap back to a healthy bell with Batch Norm

Data flows left→right through 4 layers. Each histogram is that layer's activation distribution for one mini-batch. Without BN the distribution drifts and spreads deeper into the net — many activations pile up at the edges and saturate (gradients die) — that's internal covariate shift, and the loss curve descends slowly and shakily. Flip Batch Norm on and after every layer the activations are re-centered to ~mean 0 / std 1 — each histogram snaps back to a tidy bell — so the loss drops faster and more stably, and you can crank the learning rate without it diverging. Move the learning-rate slider high and compare.

↑ Batch Norm normalizes each layer's inputs per mini-batch — subtract the batch mean, divide by the batch std — then rescales with two learnable parameters γ (scale) and β (shift) so the network can still represent any distribution it needs. Keeping activations in a healthy range stops them drifting and saturating layer after layer, which makes deeper networks train faster and far more stably, tolerates much higher learning rates, and adds a little regularization for free.

🔨 Batch Norm, in code

    
Take a layer's pre-activations for a mini-batch → compute the batch mean & variance per feature → normalize to mean 0 / std 1 → apply the learnable scale γ and shift β → keep a running mean & variance so inference doesn't need a batch → and in real code just drop in nn.BatchNorm1d / 2d (or LayerNorm for Transformers). ~7 steps of plain Python — the whole transform is four short lines.

    
    
      
  1. 1
    Start from a layer's pre-activations for one mini-batch
    
        
    import numpy as np

# x is the output of a linear layer for a whole mini-batch:
#   shape (batch_size, num_features)
# Each COLUMN is one feature across all examples in the batch.
batch_size, num_features = 32, 64
x = np.random.randn(batch_size, num_features) * 3 + 1.5   # drifted + spread

gamma = np.ones(num_features)    # learnable scale  (start = 1)
beta  = np.zeros(num_features)   # learnable shift  (start = 0)
eps   = 1e-5                     # avoids divide-by-zero
    
          
    2. 

      
  2. 2
    Compute the batch mean & variance — per feature
    
        
    # axis=0 -> reduce over the BATCH, keeping one stat per feature.
mu  = x.mean(axis=0)             # shape (num_features,)
var = x.var(axis=0)              # shape (num_features,)

# These describe THIS mini-batch's distribution for each feature:
# where it's centered (mu) and how spread out it is (var).
    
          
    3. 

      
  3. 3
    Normalize to mean 0 / std 1
    
        
    # subtract the mean, divide by the std (sqrt of var + eps):
x_hat = (x - mu) / np.sqrt(var + eps)

# now every feature column has ~mean 0 and ~std 1 for this batch.
# this is the core trick — activations can't drift or blow up.
print(x_hat.mean(axis=0).round(3))   # ~ all zeros
print(x_hat.std(axis=0).round(3))    # ~ all ones
    
          
    4. 

      
  4. 4
    Apply the learnable scale γ and shift β
    
        
    # Forcing mean 0 / std 1 might be too restrictive, so let the network
# learn to rescale & re-shift. gamma and beta are trained by backprop.
out = gamma * x_hat + beta

# If gamma -> std and beta -> mean, BN can perfectly UNDO itself.
# So BN never costs the network expressive power; it only HELPS optimisation.
    
          
    5. 

      
  5. 5
    Track running stats for inference
    
        
    # At inference you may have ONE example -> no batch to take stats from.
# So during training keep an exponential moving average of mu and var:
momentum = 0.1
running_mean = (1 - momentum) * running_mean + momentum * mu
running_var  = (1 - momentum) * running_var  + momentum * var

# at TEST time, normalize with the running stats (NOT the batch's):
x_hat_test = (x - running_mean) / np.sqrt(running_var + eps)
out_test   = gamma * x_hat_test + beta
    
          
    6. 

      
  6. 6
    The whole thing as one reusable function
    
        
    def batch_norm(x, gamma, beta, running_mean, running_var,
               training=True, momentum=0.1, eps=1e-5):
    if training:
        mu, var = x.mean(0), x.var(0)
        running_mean = (1-momentum)*running_mean + momentum*mu
        running_var  = (1-momentum)*running_var  + momentum*var
    else:
        mu, var = running_mean, running_var      # use the saved stats
    x_hat = (x - mu) / np.sqrt(var + eps)         # normalize
    return gamma * x_hat + beta, running_mean, running_var   # scale + shift

  7. 7
    In real code: nn.BatchNorm & LayerNorm
    
        
    import torch.nn as nn

# --- PyTorch does all of the above for you ---
net = nn.Sequential(
    nn.Linear(64, 128), nn.BatchNorm1d(128), nn.ReLU(),  # 1d: tabular/MLP
    nn.Linear(128, 10),
)
# for images use BatchNorm2d(channels) after a Conv2d.
# model.train() uses batch stats; model.eval() uses running stats.

# --- LayerNorm: normalize across FEATURES of ONE sample (no batch needed) ---
ln = nn.LayerNorm(128)         # used in Transformers (batch-size independent)
# GroupNorm is a middle ground for small batches / vision.

🎉 Day 19 of DeepLearningFromZero done.
Batch Norm normalizes each layer's inputs per mini-batch — subtract the batch mean, divide by the batch std — then rescales with two learnable parameters γ (scale) and β (shift) so the network keeps full expressive power. This stops activations drifting and saturating across depth (internal covariate shift), so deeper nets train faster, tolerate higher learning rates, and need less careful initialization, with a bonus of mild regularization. It depends on batch size, so for tiny batches or sequence models we reach for LayerNorm (the default in Transformers) or GroupNorm — but the idea is the same: keep activations in a healthy range so optimisation stays easy.
