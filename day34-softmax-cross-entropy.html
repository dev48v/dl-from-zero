DeepLearningFromZero · Day 34

🧮 Softmax + Cross-Entropy — a network's raw scores (logits) are squashed into a probability distribution by softmax, scored against the true class by cross-entropy, and — because the two were built to fit together — the whole gradient collapses to one beautiful expression: predicted − true

Four raw scores go in. A probability distribution, a loss, and a gradient come out — live.

A classifier's last layer emits logits z — unbounded real numbers, one per class, that mean nothing on their own. Softmax turns them into probabilities: exponentiate, then normalize so they sum to 1. To measure how wrong we are, cross-entropy compares that distribution to the true label — and for a one-hot label it is simply −log(ptrue), the surprise of the correct answer. Drag the four logits, pick which class is actually correct, and watch three things update at once: the softmax probabilities (computed with the max-subtraction stability trick), the cross-entropy loss, and the gradient ∂L/∂z = p − y that would push each logit up or down.

🔨 Build it yourself

    
We build the whole pipeline from scratch in NumPy so nothing is hidden: a numerically stable softmax, cross-entropy against a one-hot label, and the combined gradient softmax − onehot — then a one-step gradient check, then the idiomatic framework version (where you pass raw logits straight to F.cross_entropy and never call softmax yourself), and finally where this exact loss powers every classifier and every token an LLM predicts.

    
    
      
  1. 
        
    1
    Stable softmax — subtract the max first
    
        
    Exponentiate the logits and normalize. The only subtlety is numerical: exp of a large logit overflows to inf. Subtracting the max before exponentiating is mathematically identical (softmax is shift-invariant) but keeps the largest exponent at exp(0)=1.
    
        
    import numpy as np

def softmax(z):
    z = z - np.max(z, axis=-1, keepdims=True)   # shift for stability
    e = np.exp(z)                                # largest term is now exp(0)=1
    return e / e.sum(axis=-1, keepdims=True)     # sums to 1

z = np.array([2.0, 1.0, 0.1, -1.5])
print(softmax(z))          # -> [0.646, 0.238, 0.097, 0.020]
print(softmax(z).sum())    # -> 1.0
print(softmax(z + 1000))   # identical: adding a constant cancels

    Cross-entropy against a one-hot label
    
        
    Cross-entropy is −Σ yᵢ log(pᵢ). When y is one-hot, every term is zero except the true class, so the loss collapses to −log(p_true) — the negative log-likelihood of the correct answer. Add a tiny eps so log(0) never blows up.
    
        
    def cross_entropy(p, y_true):          # y_true = the correct class index
    eps = 1e-12
    return -np.log(p[y_true] + eps)    # only the true class contributes

p = softmax(np.array([2.0, 1.0, 0.1, -1.5]))
print(cross_entropy(p, y_true=0))   # confident & correct -> ~0.44 (low)
print(cross_entropy(p, y_true=3))   # confident & WRONG   -> ~3.94 (high)

    The combined gradient — p − y
    
        
    Here is why the pair is inseparable. Cross-entropy of softmax has the gradient ∂L/∂zᵢ = pᵢ − yᵢ — no exponentials, no division, no chain of ugly terms. You never differentiate softmax and cross-entropy separately; you fuse them and get one subtraction: predicted distribution minus the one-hot target.
    
        
    def softmax_ce_grad(z, y_true):
    p = softmax(z)
    onehot = np.zeros_like(p)
    onehot[y_true] = 1.0
    return p - onehot          # <-- the entire gradient, dL/dz

z = np.array([2.0, 1.0, 0.1, -1.5])
print(softmax_ce_grad(z, y_true=0))
# [-0.354, 0.238, 0.097, 0.020]
#   ^ true class: negative (push logit UP); others positive (push DOWN)

    Prove it: one gradient-descent step lowers the loss
    
        
    The gradient is only useful if stepping against it reduces the loss. Take one step z ← z − lr·(p−y) and watch cross-entropy drop and p_true rise — a numerical sanity check that the clean formula is correct.
    
        
    z = np.array([2.0, 1.0, 0.1, -1.5]); y = 0; lr = 1.0
for step in range(4):
    p = softmax(z)
    loss = cross_entropy(p, y)
    print(f"step {step}: loss={loss:.3f}  p_true={p[y]:.3f}")
    z = z - lr * softmax_ce_grad(z, y)   # descend
# loss falls each step; p_true climbs toward 1.0

    The framework version — feed it raw logits
    
        
    In PyTorch you almost never call softmax before the loss. nn.CrossEntropyLoss / F.cross_entropy take the raw logits and the integer target: internally they fuse log_softmax + nll_loss using the log-sum-exp trick, which is both numerically stable and gives the exact p−y gradient for free. Applying softmax yourself first is the classic bug — it double-softmaxes and quietly hurts training.
    
        
    import torch, torch.nn.functional as F

logits = torch.tensor([[2.0, 1.0, 0.1, -1.5]], requires_grad=True)
target = torch.tensor([0])                 # the true class INDEX (not one-hot)

loss = F.cross_entropy(logits, target)     # pass RAW logits, never softmax(...)
loss.backward()
print(loss.item())        # ~0.44
print(logits.grad)        # tensor([[-0.354, 0.238, 0.097, 0.020]])  == p - onehot

# DON'T:  F.cross_entropy(F.softmax(logits, -1), target)   # double softmax bug

    Where it shows up — every classifier, every LLM token
    
        
    This is the default loss for classification everywhere. An image classifier's final logits → softmax → cross-entropy. And every word a language model predicts is a softmax over the whole vocabulary scored by cross-entropy against the actual next token — the p−y gradient is what trains GPT-scale models one token at a time. The temperature T you met on Day 33 is just softmax(z/T): the same function, dialled softer.
    
        
    # next-token prediction (the entire LLM training objective, distilled):
logits = model(context)                 # shape [vocab_size], e.g. 50k
loss   = F.cross_entropy(logits, next_token_id)
# grad on the logits = softmax(logits) - onehot(next_token_id)
# every class gets a nudge; the true token's logit is pulled up.

probs = F.softmax(logits / T, dim=-1)   # inference: T tunes creativity

🎉 Day 34 of DeepLearningFromZero done.
Softmax converts a vector of raw logits into a probability distribution — exponentiate, then normalize — and the max-subtraction trick makes it numerically safe without changing the answer, because softmax is invariant to adding a constant to every logit. Cross-entropy scores that distribution against the truth; for a one-hot label it is simply −log(ptrue), the negative log-likelihood of the correct class. The reason the two are always used together is the gradient: differentiate cross-entropy of a softmax and everything cancels into a single expression — ∂L/∂z = p − y, the predicted distribution minus the one-hot target. No exponentials survive, one learning rate works, and it is exactly what a framework's cross_entropy computes when you hand it raw logits (never pre-softmaxed). This one loss trains virtually every classifier and every next-token an LLM produces. 👉 Tomorrow, Day 35: label smoothing — softening those hard one-hot targets to curb overconfidence.
