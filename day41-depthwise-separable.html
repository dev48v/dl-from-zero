DeepLearningFromZero · Day 41

📱 Depthwise-Separable Convolutions — a standard conv does two jobs at once with one fat tensor (Dk×Dk×Cin×Cout): it filters space and mixes channels in a single sweep, paying Dk²·Cin·Cout multiply-adds per pixel. MobileNet factors that into two cheap steps. DEPTHWISE: one Dk×Dk filter per input channel — pure spatial filtering, Cin→Cin, zero cross-channel mixing (Dk²·Cin). POINTWISE: a 1×1 conv that mixes channels at every pixel with no spatial extent, Cin→Cout (Cin·Cout). Same output shape, but the cost drops by 1/Cout + 1/Dk² — roughly 8–9× fewer multiplies for a 3×3 kernel — for a tiny accuracy hit. It is the engine of MobileNet, Xception and every on-device CNN.

Drag the four dimensions and watch the multiply-add counts — and the savings ratio — recompute live.

A standard convolution turns Cin input channels into Cout output channels over a Df×Df feature-map using a Dk×Dk kernel — and it costs Dk²·Cin·Cout·Df² multiply-adds. The depthwise-separable version splits that into a per-channel spatial depthwise conv plus a 1×1 channel-mixing pointwise conv. Notice the ratio does not depend on Df (it cancels), and that a 1×1 kernel gives no saving at all.

We assemble the whole idea one line at a time: the standard conv cost we are trying to beat, a from-scratch depthwise conv (per-channel spatial filter), the pointwise 1×1 mixer, the two glued into one separable block with BN + ReLU, the reduction-ratio arithmetic, the real groups=Cin trick in PyTorch, MobileNet’s width & resolution multipliers, and finally the one import you actually use.

    
    
      
  1. 
        
    1
    The baseline — what a standard conv actually costs
    
        
    A standard conv slides a Dk×Dk×Cin filter Cout times over the map. For every one of the Df² output pixels and every one of the Cout outputs it multiplies Dk²·Cin values — so it fuses spatial filtering and channel mixing into a single, expensive operation.
    
        
    // standard conv: multiply-adds for a Df x Df output
function stdCost(Dk, Cin, Cout, Df) {
  return Dk*Dk * Cin * Cout * Df*Df;   // one fat tensor does everything
}
// weights (params) alone:  Dk*Dk * Cin * Cout
// e.g. 3x3, Cin=Cout=512  ->  9*512*512 = 2,359,296 weights per layer

  2. 
        
    2
    Depthwise — one spatial filter per channel, no mixing
    
        
    Give each input channel its own Dk×Dk kernel and convolve it in isolation. Channel c in → channel c out; nothing crosses between channels. The output has the same Cin channels, and the cost collapses to Dk²·Cin·Df² — the Cout factor is gone.
    
        
    // depthwise: conv each channel with its OWN kernel (no mixing)
function depthwise(maps, kernels, Dk) {   // maps[c], kernels[c] = Dk*Dk
  return maps.map((m, c) => conv2d(m, kernels[c]));  // c -> c only
}
function dwCost(Dk, Cin, Df) { return Dk*Dk * Cin * Df*Df; }
// output still has Cin channels; pure spatial filtering.

  3. 
        
    3
    Pointwise — a 1×1 conv that mixes channels
    
        
    Depthwise never combined channels, so a 1×1 conv does exactly that. At each pixel it takes a weighted sum across all Cin channels to produce each of Cout outputs — a plain matrix multiply per pixel, no spatial extent. Cost Cin·Cout·Df².
    
        
    // pointwise: 1x1 conv = per-pixel channel mix  (Cin -> Cout)
function pointwise(chVals, W) {        // chVals: length Cin at one pixel
  return W.map(row =>                  // W: [Cout][Cin]
    row.reduce((a, w, c) => a + w * chVals[c], 0));
}
function pwCost(Cin, Cout, Df) { return Cin*Cout * Df*Df; }
// no spatial footprint (1x1) — this is where new features are built.

  4. 
        
    4
    The separable block — depthwise then pointwise
    
        
    Chain them: depthwise for where, pointwise for what. In MobileNet each conv is followed by BatchNorm + ReLU, so the block is DW → BN → ReLU → PW → BN → ReLU. Same input and output shapes as one standard conv — it is a drop-in replacement, just far cheaper.
    
        
    // depthwise-separable = depthwise (space) THEN pointwise (channels)
function separable(maps, dwKernels, pwW, Dk) {
  const spatial = depthwise(maps, dwKernels, Dk);  // Cin -> Cin
  return pointwiseAllPixels(spatial, pwW);         // Cin -> Cout
}
// MobileNet block:  DW -> BN -> ReLU -> PW(1x1) -> BN -> ReLU
// identical in/out shape to one standard conv, a fraction of the cost.

    The reduction ratio — where 8–9× comes from
    
        
    Divide separable cost by standard cost and Df² and one Cin cancel, leaving 1/Cout + 1/Dk². With a 3×3 kernel and a decent channel count the second term dominates: ≈ 1/9, so about 8–9× cheaper. A 1×1 kernel gives 1/Cout + 1 — no saving, because there is no spatial cost to factor out.
    
        
    function reduction(Dk, Cout) {         // separable / standard
  return 1/Cout + 1/(Dk*Dk);
}
// Dk=3, Cout=256 -> 1/256 + 1/9 = 0.115  ->  ~8.7x fewer multiplies
// Dk=1           -> 1/Cout + 1   > 1      ->  NO benefit (1x1 already)
// note: independent of Df and (mostly) of Cin.

    In PyTorch — the groups=Cₓ trick
    
        
    There is no separate “depthwise” layer — you get it by setting groups=in_channels on a normal Conv2d, which forces each channel into its own group with no mixing. The pointwise step is just a Conv2d with kernel_size=1. Two lines rebuild the whole block.
    
        
    import torch.nn as nn

class SeparableConv(nn.Module):
    def __init__(self, Cin, Cout, Dk=3, stride=1):
        super().__init__()
        # depthwise: groups=Cin  => one filter per channel, no mixing
        self.dw = nn.Conv2d(Cin, Cin, Dk, stride, Dk//2,
                            groups=Cin, bias=False)
        self.pw = nn.Conv2d(Cin, Cout, 1, bias=False)   # pointwise 1x1
        self.bn1, self.bn2 = nn.BatchNorm2d(Cin), nn.BatchNorm2d(Cout)
    def forward(self, x):
        x = torch.relu(self.bn1(self.dw(x)))   # space
        return torch.relu(self.bn2(self.pw(x)))# channels

    MobileNet’s two dials — width α and resolution ρ
    
        
    On top of the factorization, MobileNet exposes two global shrink knobs. The width multiplier α scales every channel count (cost ∝ α²); the resolution multiplier ρ scales the input size (cost ∝ ρ²). Together they sweep out a whole family of models trading accuracy for latency — MobileNet-0.5-160, etc.
    
        
    def scaled_cost(Dk, Cin, Cout, Df, alpha=1.0, rho=1.0):
    Cin, Cout = int(Cin*alpha), int(Cout*alpha)   # width  ~ alpha^2
    Df = int(Df*rho)                              # resolution ~ rho^2
    return Dk*Dk*Cin*Df*Df + Cin*Cout*Df*Df       # depthwise + pointwise
# alpha in {1.0, 0.75, 0.5, 0.25}, rho set by input {224,192,160,128}
# alpha=0.5 -> ~4x cheaper still, a few % accuracy for big speedups.

    The real thing — one import, and the family it built
    
        
    You never hand-roll it. torchvision ships MobileNetV2/V3 pretrained; timm has them plus Xception and EfficientNet (which pairs the separable block with Day-40’s squeeze-excite). The depthwise-separable idea is now the default building block for anything that must run on a phone.
    
        
    import torchvision.models as m
net = m.mobilenet_v3_large(weights="DEFAULT")   # separable convs everywhere

import timm
net = timm.create_model('mobilenetv3_small_100', pretrained=True)
net = timm.create_model('xception', pretrained=True)   # separable, deeper
# lineage: MobileNetV1 (this) -> V2 inverted residuals -> V3 + SE
#          -> EfficientNet (separable + squeeze-excite + compound scaling)

🎉 Day 41 of DeepLearningFromZero done.
Depthwise-separable convolutions factor one expensive standard conv into two cheap ones. A depthwise conv applies a single Dk×Dk spatial filter to each channel independently (Cin→Cin, cost Dk²·Cin·Df²) with no cross-channel mixing; a pointwise 1×1 conv then mixes all channels at every pixel (Cin→Cout, cost Cin·Cout·Df²) with no spatial extent. Same output shape as a standard conv, but the multiply-adds drop by 1/Cout + 1/Dk² — roughly 8–9× for a 3×3 kernel — for only a small accuracy cost. In PyTorch you get depthwise for free via groups=Cin, and MobileNet layers a width multiplier α and resolution multiplier ρ on top to shrink further. It powers MobileNet, Xception, and (with squeeze-excite) EfficientNet. 👉 Tomorrow, Day 42: EfficientNet & compound scaling — balance depth, width and resolution with one coefficient, stacking MBConv blocks (inverted-residual depthwise-separable + squeeze-excite) into a state-of-the-art efficiency frontier.
