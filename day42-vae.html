DeepLearningFromZero · Day 42

🎲 Variational Autoencoder — a plain autoencoder (Day 16) squeezes each input to a single point in a latent code and decodes it back. That works for compression, but the latent space it learns is holey: training points sit on islands with dead gaps between them, so if you pick a random code — or a point between two examples — the decoder produces garbage. You cannot sample from it, so you cannot generate. A VAE fixes this by making the encoder output a distribution instead of a point: for every latent dimension it emits a mean μ and a log-variance logσ², defining a little Gaussian N(μ,σ²). You then sample a code z ~ N(μ,σ) and decode that — forcing a whole neighbourhood to decode sensibly. Sampling is not differentiable, so the reparameterization trick rewrites it as z = μ + σ⊙ε, ε~N(0,1), moving the randomness into ε so gradients still flow to μ and σ. The loss adds a KL term — recon + β·KL(N(μ,σ)‖N(0,1)) — that pulls every posterior toward a standard normal, gluing the islands into one continuous, samplable space you can draw new data from and interpolate smoothly across.

🔨 Build it yourself

    
A VAE is a plain autoencoder plus three additions: the encoder emits a mean and a log-variance instead of a code; the reparameterization trick turns sampling into a differentiable expression; and the loss gains a closed-form Gaussian KL that pulls the posterior toward N(0,1). We assemble it one line at a time — the two encoder heads, the reparam sampler, the decoder, the reconstruction loss, the KL one-liner, the combined β-weighted loss (the negative ELBO), a training step, generation from the prior, and finally the real nn.Module.

    
    
      
  1. 
        
    1
    The encoder head — output μ and logσ², not a code
    
        
    A normal encoder ends in one linear layer → the code. A VAE encoder ends in two linear layers over the same hidden features: one produces the mean μ, the other the log-variance logσ² (we predict the log so σ² is always positive and unconstrained). Together they define a diagonal Gaussian posterior q(z|x)=N(μ,σ²) for this input.
    
        
    import torch, torch.nn as nn

class Encoder(nn.Module):
    def __init__(self, in_dim, hidden, zdim):
        super().__init__()
        self.body    = nn.Sequential(nn.Linear(in_dim, hidden), nn.ReLU())
        self.fc_mu   = nn.Linear(hidden, zdim)   # mean  of q(z|x)
        self.fc_logv = nn.Linear(hidden, zdim)   # LOG-variance (unconstrained)
    def forward(self, x):
        h = self.body(x)
        return self.fc_mu(h), self.fc_logv(h)    # mu, logvar  (each [B, zdim])

  2. 
        
    2
    The reparameterization trick — make sampling differentiable
    
        
    We must sample z ~ N(μ,σ²), but drawing a random sample has no gradient w.r.t. μ/σ — backprop would stop dead. The trick: sample the randomness outside the parameters as ε~N(0,1), then form z = μ + σ⊙ε. Now z is a smooth, differentiable function of μ and σ (with σ=exp(½logσ²)), and ∂z/∂μ=1, ∂z/∂σ=ε, so gradients flow through.
    
        
    def reparameterize(mu, logvar):
    std = torch.exp(0.5 * logvar)      # sigma = exp(0.5 * log sigma^2)
    eps = torch.randn_like(std)        # epsilon ~ N(0, 1), the randomness
    return mu + std * eps              # z = mu + sigma (elementwise) eps
# gradient still flows to mu (dz/dmu = 1) and std (dz/dstd = eps).

  3. 
        
    3
    The decoder — a code back to data
    
        
    Identical in spirit to an autoencoder decoder: map the sampled z back to the input space. For binary/greyscale pixels end in a sigmoid so outputs are in [0,1] and can be read as Bernoulli probabilities per pixel.
    
        
    class Decoder(nn.Module):
    def __init__(self, zdim, hidden, out_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(zdim, hidden), nn.ReLU(),
            nn.Linear(hidden, out_dim), nn.Sigmoid())  # -> [0,1] per pixel
    def forward(self, z):
        return self.net(z)             # x_hat, same shape as the input

  4. 
        
    4
    Reconstruction loss — did we get the input back?
    
        
    The first half of the loss is the usual autoencoder term: how far the reconstruction is from the input. For pixels in [0,1] use binary cross-entropy summed over pixels (the Bernoulli log-likelihood); for continuous data use MSE. Summed per sample, then averaged over the batch.
    
        
    import torch.nn.functional as F
def recon_loss(x_hat, x):
    # BCE per pixel, summed over pixels (Bernoulli neg-log-likelihood)
    return F.binary_cross_entropy(x_hat, x, reduction='sum') / x.size(0)
# continuous data instead:  F.mse_loss(x_hat, x, reduction='sum')/B

    The Gaussian KL — one closed-form line
    
        
    Because both the posterior N(μ,σ²) and the prior N(0,1) are Gaussians, the KL divergence has an exact closed form — no sampling needed: KL = ½ Σi (μi² + σi² − 1 − logσi²). It is zero only when every μ=0 and σ=1 (exactly the prior), and grows as the posterior drifts away — so minimizing it pulls each blob toward the origin with unit spread.
    
        
    def kl_divergence(mu, logvar):
    # KL( N(mu, sigma^2) || N(0,1) ), summed over latent dims, per sample
    # = 0.5 * sum( mu^2 + sigma^2 - 1 - log sigma^2 )
    return -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp()) / mu.size(0)
# note: 1 + logvar - mu^2 - e^logvar  ==  -(mu^2 + sigma^2 - 1 - log sigma^2)

    The full VAE loss — recon + β·KL (the negative ELBO)
    
        
    Add the two: L = recon + β·KL. This is exactly the negative ELBO (evidence lower bound) — minimizing it maximizes a lower bound on log p(x). β=1 is the original VAE; β>1 (a β-VAE) weights the prior harder for a more continuous, disentangled space at some cost to sharpness; β=0 collapses back to a plain autoencoder.
    
        
    def vae_loss(x, x_hat, mu, logvar, beta=1.0):
    rec = recon_loss(x_hat, x)
    kl  = kl_divergence(mu, logvar)
    return rec + beta * kl,  rec, kl     # total = negative ELBO
# beta = 1 -> standard VAE ;  beta > 1 -> beta-VAE ;  beta = 0 -> plain AE

    A training step — and generating new data
    
        
    Forward: encode → reparameterize → decode → loss → backward. Because KL pushed the aggregate posterior toward N(0,1), generation is trivial: sample z~N(0,1) from the prior and decode — no encoder, no input needed. That is the whole payoff over a plain autoencoder.
    
        
    def train_step(x, enc, dec, opt, beta=1.0):
    mu, logvar = enc(x)
    z          = reparameterize(mu, logvar)
    x_hat      = dec(z)
    loss, rec, kl = vae_loss(x, x_hat, mu, logvar, beta)
    opt.zero_grad(); loss.backward(); opt.step()
    return loss.item()

@torch.no_grad()                          # GENERATE brand-new samples:
def generate(dec, n, zdim):
    z = torch.randn(n, zdim)              # z ~ prior N(0,1)
    return dec(z)                         # decode -> new data

    The real thing — one module, and where it goes next
    
        
    Glue the pieces into one nn.Module. The VAE is the ancestor of a whole family: conditional VAEs (label-conditioned generation), VQ-VAE (a discrete codebook — the tokenizer behind modern image models), and latent-diffusion (Day 15 diffusion run inside a VAE’s latent space, which is how Stable Diffusion works). The probabilistic-latent idea you built here is everywhere.
    
        
    class VAE(nn.Module):
    def __init__(self, in_dim=784, hidden=400, zdim=20):
        super().__init__()
        self.enc = Encoder(in_dim, hidden, zdim)
        self.dec = Decoder(zdim, hidden, in_dim)
    def forward(self, x):
        mu, logvar = self.enc(x)
        z = reparameterize(mu, logvar)
        return self.dec(z), mu, logvar

# lineage:  VAE -> beta-VAE -> conditional VAE -> VQ-VAE (discrete codes)
#           -> latent diffusion (Stable Diffusion decodes in a VAE latent)

🎉 Day 42 of DeepLearningFromZero done.
A variational autoencoder turns the deterministic code of a plain autoencoder into a probabilistic one. The encoder outputs a mean μ and log-variance logσ² per latent dim, defining a Gaussian posterior q(z|x)=N(μ,σ²); we draw z from it via the reparameterization trick z=μ+σ⊙ε, ε~N(0,1) so gradients still flow. The loss is recon + β·KL(N(μ,σ)‖N(0,1)) — the negative ELBO — where the closed-form Gaussian KL ½Σ(μ²+σ²−1−logσ²) pulls every posterior toward the standard-normal prior, gluing the latent space into one continuous, samplable region. That is what lets you generate new data by decoding z~N(0,1) and interpolate smoothly between examples — things a plain autoencoder’s holey latent simply cannot do. 👉 Tomorrow, Day 43: VQ-VAE — swap the continuous Gaussian latent for a discrete learned codebook (vector quantization), the trick that turns images into tokens for autoregressive and diffusion generators.
