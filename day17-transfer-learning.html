↑ A model pretrained on millions of images already learned general features — edges, textures, shapes. Fine-tuning unfreezes a few top layers with a tiny learning rate to squeeze out a bit more — at the cost of more trainable parameters.

🔨 Transfer learning, in code

    
Load a model pretrained on a huge dataset → freeze its backbone so those general features stay put → replace the head with a small new one for your task → train just the head on your few examples → optionally unfreeze the top layers and fine-tune with a very low learning rate. ~7 steps in PyTorch (Keras + Hugging Face shown too). Every line is the real thing — transfer learning is mostly turning gradients off and swapping the last layer.

    
    
      
  1. 1
    Load a model pretrained on a huge dataset
    
        
    import torch, torch.nn as nn
from torchvision import models

# ResNet-18 with weights already trained on ImageNet (1.2M images, 1000 classes)
model = models.resnet18(weights=models.ResNet18_Weights.IMAGENET1K_V1)
# the convolutional "backbone" already knows edges, textures, shapes, parts.
# we did NOT train any of this — we downloaded it.
    
          
    2. 

      
  2. 2
    Freeze the backbone — turn its gradients off
    
        
    # stop the backbone from updating: its weights stay exactly as pretrained
for param in model.parameters():
    param.requires_grad = False
# with requires_grad=False, backprop skips these layers entirely.
# this is feature extraction: reuse the frozen features as-is.
    
          
    3. 

      
  3. 3
    Replace the head with one for YOUR task
    
        
    # the pretrained head predicts 1000 ImageNet classes — useless for us.
# swap it for a small new layer sized to OUR number of classes.
NUM_CLASSES = 3                       # e.g. cat / dog / rabbit
in_feats = model.fc.in_features       # 512 for resnet18
model.fc = nn.Linear(in_feats, NUM_CLASSES)
# the new fc layer is created fresh -> requires_grad=True by default,
# so ONLY this tiny head will train.
    
          
    4. 

      
  4. 4
    Optimise ONLY the trainable parameters
    
        
    # hand the optimiser only the params that still require gradients (the head)
trainable = [p for p in model.parameters() if p.requires_grad]
opt = torch.optim.Adam(trainable, lr=1e-3)
loss_fn = nn.CrossEntropyLoss()
# far fewer params to learn -> needs far less data and far less time.
    
          
    5. 

      
  5. 5
    Train the head on your few examples
    
        
    for epoch in range(epochs):
    for x, y in your_small_dataloader:     # maybe only a few hundred images
        logits = model(x)                  # backbone (frozen) -> head (learns)
        loss = loss_fn(logits, y)
        opt.zero_grad(); loss.backward(); opt.step()
    print(epoch, loss.item())
# because the backbone is frozen, training is fast and won't overfit your data.
    
          
    6. 

      
  6. 6
    Optional: unfreeze the top + fine-tune (low LR)
    
        
    # once the head is trained, gently adapt the TOP of the backbone too.
for param in model.layer4.parameters():    # just the last block
    param.requires_grad = True
# CRUCIAL: a tiny learning rate, so we nudge — not wreck — the good features
opt = torch.optim.Adam(
    [p for p in model.parameters() if p.requires_grad], lr=1e-5)
# too-high LR here causes "catastrophic forgetting" of the pretrained knowledge.
    
          
    7. 

      
  7. 7
    Same idea in Keras & Hugging Face
    
        
    # --- Keras (vision) ---
base = keras.applications.ResNet50(include_top=False, weights="imagenet")
base.trainable = False                       # freeze backbone
model = keras.Sequential([base,
    keras.layers.GlobalAveragePooling2D(),
    keras.layers.Dense(NUM_CLASSES, activation="softmax")])   # new head

# --- Hugging Face (text/LLMs) — same recipe, different domain ---
from transformers import AutoModelForSequenceClassification
m = AutoModelForSequenceClassification.from_pretrained(
        "bert-base-uncased", num_labels=NUM_CLASSES)   # pretrained backbone + fresh head
# fine-tune on your labelled text with a low LR (e.g. 2e-5).
# this is literally how fine-tuning an LLM works.

🎉 Day 17 of DeepLearningFromZero done.
Don't start from zero — stand on a model that already learned the world. Load a pretrained backbone, freeze it so its general features (edges, textures, shapes) stay intact, swap on a small new head, and train just that on your handful of examples. Need a little more? Unfreeze the top layers and fine-tune with a tiny learning rate. It's why ImageNet backbones, BERT, and foundation models are everywhere — and exactly why fine-tuning an LLM works.
