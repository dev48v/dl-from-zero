DeepLearningFromZero · Day 40

🎚️ Squeeze-and-Excitation — a plain convolution treats all C output channels as equally important. SE bolts on a lightweight channel-attention gate. SQUEEZE: collapse each channel’s H×W feature-map to a single number with global average pooling — a per-channel descriptor. EXCITE: push that C-vector through a tiny two-layer bottleneck MLP (FC C→C/r → ReLU → FC C/r→C → sigmoid) to get per-channel gates in (0,1). SCALE: multiply every feature-map by its gate. The net learns which channels matter for this input and recalibrates them — amplifying the informative ones, damping the rest — for only ≈2C²/r extra params. It won ILSVRC 2017 and now lives inside MobileNetV3 and EfficientNet.

Pick an input, then watch squeeze → excite → rescale run for real. Drag r and see the gates — and the cost — change.

Below are C = 8 feature-maps (each a 4×4 grid) produced by a conv layer. The squeeze averages each whole map down to one number zc. The excite MLP reads that 8-vector, funnels it through a bottleneck of width C/r, and emits a gate sc∈(0,1) per channel. Finally we rescale: each map is multiplied by its gate, so suppressed channels fade to near-nothing and boosted ones stay vivid. Everything here — the pooling, the two FC layers, the ReLU and sigmoid — is computed live in your browser. 🔨 Build it yourself

    
The live demo is a from-scratch SE block — no library, just an average, two matrix multiplies, a ReLU and a sigmoid. We assemble it one line at a time: the squeeze (global average pool), the two-layer excite MLP with its reduction bottleneck, the sigmoid gates, the channel-wise scale, the whole thing as a tidy nn.Module, the r-vs-cost arithmetic, how it drops into a ResNet bottleneck, and finally the one line you actually write in practice.

    
    
      
  1. 
        
    1
    Squeeze — global average pool each channel to one number
    
        
    Take the C×H×W feature-map and average over the whole spatial plane, giving a C-length vector z. Element zc is a scalar summary of channel c with a global receptive field — the network can now reason about a channel using the entire image, not just a 3×3 window.
    
        
    // squeeze: average every HxW map down to a single value
function squeeze(maps) {            // maps: C channels, each H*W values
  return maps.map(m =>              // -> z: one number per channel
    m.reduce((a, v) => a + v, 0) / m.length);
}
// z has length C; z[c] = mean activation of channel c  (a descriptor)

  2. 
        
    2
    Excite, part 1 — squeeze the descriptor through a bottleneck
    
        
    First FC layer maps the C-vector down to C/r hidden units, then ReLU. The reduction ratio r (paper default 16) is the whole point: the bottleneck is far cheaper than a full C→C layer and forces the block to capture channel relationships rather than memorise each channel independently.
    
        
    function fc(vec, W, b) {                       // W: [out][in], b: [out]
  return W.map((row, i) =>
    row.reduce((a, w, j) => a + w * vec[j], 0) + b[i]);
}
const relu = v => v.map(x => Math.max(0, x));

// C -> C/r  (the reduction bottleneck), then ReLU
const hidden = relu(fc(z, W1, b1));            // length C/r
    
      
    3. 
      
  3. 
        
    3
    Excite, part 2 — expand back to C and sigmoid into gates
    
        
    Second FC layer maps C/r → C, then a sigmoid squashes each output into (0,1). These are the gates s. Sigmoid (not softmax) is deliberate: channels are not mutually exclusive — many can be “on” at once — so each gets its own independent 0–1 importance.
    
        
    const sigmoid = v => v.map(x => 1 / (1 + Math.exp(-x)));

// C/r -> C, then sigmoid -> per-channel gates in (0,1)
const s = sigmoid(fc(hidden, W2, b2));         // length C
// s[c] near 1 -> keep channel c ; near 0 -> suppress it
// sigmoid, NOT softmax: gates are independent, not competing.
    
      
    4. 
      
  4. 
        
    4
    Scale — recalibrate each map by its gate
    
        
    Multiply every value of channel c by its scalar gate sc (broadcast across H×W). That is the entire “recalibration”: the block re-weights the channels of its own input and passes the result on. Cheap element-wise multiply, big representational effect.
    
        
    // scale: broadcast each gate over its whole HxW map
function scale(maps, s) {
  return maps.map((m, c) => m.map(v => v * s[c]));
}
const out = scale(maps, s);        // recalibrated feature-maps
// full block:  X~ = scale( X, excite( squeeze(X) ) )

    The whole block as one nn.Module
    
        
    In PyTorch it is a handful of lines: AdaptiveAvgPool2d(1) is the squeeze, two 1×1 convs (or Linear layers) are the excite, sigmoid makes the gates, and a broadcast multiply is the scale. Note the shapes: gates come out as (B, C, 1, 1) so they broadcast cleanly over the feature-maps.
    
        
    import torch, torch.nn as nn

class SEBlock(nn.Module):
    def __init__(self, C, r=16):
        super().__init__()
        self.squeeze = nn.AdaptiveAvgPool2d(1)      # C x H x W -> C x 1 x 1
        self.excite = nn.Sequential(
            nn.Linear(C, C // r, bias=False), nn.ReLU(inplace=True),
            nn.Linear(C // r, C, bias=False), nn.Sigmoid())
    def forward(self, x):                            # x: (B, C, H, W)
        b, c, _, _ = x.shape
        z = self.squeeze(x).view(b, c)               # (B, C)  squeeze
        s = self.excite(z).view(b, c, 1, 1)          # (B, C, 1, 1) gates
        return x * s                                 # scale (broadcast)

    The reduction ratio r — capacity vs. cost
    
        
    The two FC layers hold C·(C/r) + (C/r)·C = 2C²/r weights — the only parameters SE adds. Small r = a wider bottleneck = more capacity but more params; large r = cheaper but the gates get coarser (push it too far and they collapse toward a flat 0.5, doing nothing). The paper found r = 16 a sweet spot across channel counts.
    
        
    function seParams(C, r) { return 2 * C * (C / r); }   // = 2C^2 / r
// C=512, r=16 -> 2*512*32 = 32,768 weights for a whole stage's block
// that is a fraction of one 3x3 conv (C*C*9 = 2.36M at C=512)
// r up   -> fewer params, coarser gating
// r down -> more params, finer gating (r=1 is the full 2C^2)

    Where it slots in — SE-ResNet
    
        
    SE is a drop-in module, not a new architecture. In a ResNet bottleneck you apply it to the residual branch right before the identity is added back — the block recalibrates F(x), then y = x + SE(F(x)). The skip connection is untouched, so nothing about training stability changes; you just gained learned channel attention.
    
        
    class SEResidualBottleneck(nn.Module):
    def __init__(self, C, r=16):
        super().__init__()
        self.conv = nn.Sequential(...)     # the usual 1x1 -> 3x3 -> 1x1
        self.se   = SEBlock(C, r)          # <-- recalibrate the residual
    def forward(self, x):
        f = self.conv(x)                   # F(x)
        f = self.se(f)                     # gate F(x) channel-wise
        return torch.relu(x + f)           # add identity, THEN activate
# SE also drops into Inception, MobileNetV3, EfficientNet the same way.

    The real thing — one import, and the family it spawned
    
        
    You rarely hand-roll it. timm ships SE-ResNets and every EfficientNet (which uses SE in every block) pretrained. SE’s “recalibrate the channels” idea kicked off a whole attention lineage: CBAM adds a spatial map, ECA replaces the FC bottleneck with a cheap 1-D conv, and SE is a standard ingredient in MobileNetV3.
    
        
    import timm
net = timm.create_model('seresnet50', pretrained=True)      # SE-ResNet-50
net = timm.create_model('efficientnet_b0', pretrained=True) # SE in every block

# torchvision exposes the squeeze-excite layer directly:
from torchvision.ops import SqueezeExcitation
se = SqueezeExcitation(input_channels=256, squeeze_channels=16)  # C, C/r
# lineage of channel attention: SE -> CBAM (spatial too) -> ECA (1-D conv).

🎉 Day 40 of DeepLearningFromZero done.
Squeeze-and-Excitation gives a network cheap, learned channel attention. Squeeze collapses each channel’s H×W map to one number via global average pooling; excite runs that C-vector through a two-layer bottleneck MLP (FC C→C/r → ReLU → FC C/r→C → sigmoid) to produce a per-channel gate in (0,1); scale multiplies every map by its gate to recalibrate the stack. The only added parameters are the two FC layers, ≈2C²/r — the reduction ratio r trades capacity for cost (paper default 16). Sigmoid, not softmax, keeps the gates independent so many channels can stay on. It drops into ResNet, Inception, MobileNetV3 and EfficientNet with no retuning, won ILSVRC 2017, and seeded the attention lineage SE → CBAM → ECA. 👉 Tomorrow, Day 41: Depthwise-separable convolutions & MobileNet — factor a conv into a per-channel spatial filter plus a 1×1 mixer for order-of-magnitude fewer FLOPs.
