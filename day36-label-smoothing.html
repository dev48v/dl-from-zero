DeepLearningFromZero · Day 36

🫧 Label Smoothing — stop training on absolute 0/1 targets: replace the hard one-hot label with a soft one — (1−ε) on the true class, ε spread over the rest — so the network is pushed to be confident but not certain, which curbs overconfidence, calibrates its probabilities, and often generalises better

One example, one slider. Watch the target melt — and the confidence follow.

A classifier ends in a softmax and is trained by cross-entropy against a one-hot target: 1 on the right class, 0 everywhere else. To make softmax output exactly 1.0, the winning logit must run off to +∞ — so gradient descent never stops inflating confidence. The model gets the class right but the probability wildly wrong (“99.9% cat” even when unsure). Label smoothing gives the true class a reachable goal, 1−ε+ε/K, and a small floor ε/K under every other class. Drag ε below and watch the target soften, the loss and gradient change, and the confidence the model can converge to drop toward its real accuracy.

Calibration sketch — confidence vs. reality

A model is calibrated when its confidence matches how often it’s right. Label smoothing drags the confidence ceiling (1−ε+ε/K) down toward the true held-out accuracy. Too little ε → overconfident; too much → underconfident. (Accuracy fixed at an illustrative 85%.)

🔨 Build it yourself

    
The live demo runs entirely from scratch — no library, just a numeric softmax, a target vector, and cross-entropy. We build it one line at a time: a stable softmax, the hard one-hot target, the smoothed target from its two-line formula, the cross-entropy loss both ways, and the p − t gradient that shows why smoothing caps confidence — then we map it to the one line you actually write in PyTorch.

    
    
      
  1. 
        
    1
    One example: K classes, its logits, its true label
    
        
    Everything downstream is just arithmetic on these three things. The logits are a model’s raw scores for one input; the true class is an index. Our toy example is a nicely over-confident one — the cat logit dwarfs the rest.
    
        
    const K       = 6;                    // number of classes
const TRUE    = 0;                    // index of the correct class (cat)
const logits  = [4.5, 0.8, 1.0, 0.3, 0.5, 0.2];  // raw model scores
//               cat  dog bird car  plane ship

  2. 
        
    2
    Stable softmax — logits → probabilities
    
        
    Softmax exponentiates and normalises. Subtract the max first: it cancels out of the ratio but stops exp() from overflowing. The result sums to 1 — the model’s predicted distribution p.
    
        
    function softmax(z) {
  const m  = Math.max(...z);                 // shift for numerical stability
  const ex = z.map(v => Math.exp(v - m));
  const s  = ex.reduce((a, b) => a + b, 0);
  return ex.map(v => v / s);                  // sums to 1
}
const p = softmax(logits);   // ~[0.91, 0.02, 0.03, 0.01, 0.02, 0.01]

    3
    The hard target — a spike the softmax can never reach
    
        
    The classic one-hot label: 1 on the truth, 0 elsewhere. The catch: softmax only outputs a clean 1.0 when the true logit is infinitely bigger than the rest, so training never stops inflating confidence.
    
        
    function oneHot(K, i) {
  return Array.from({ length: K }, (_, j) => (j === i ? 1 : 0));
}
const hard = oneHot(K, TRUE);   // [1, 0, 0, 0, 0, 0]
// goal p_true = 1.0  =>  logit_true - others -> +Infinity  (unreachable)
    
      
    4. 
      
  4. 
        
    4
    The smoothed target — the whole trick, in two lines
    
        
    Blend the one-hot with a uniform distribution: y = (1−ε)·onehot + ε/K. The true class becomes 1−ε+ε/K (reachable!) and every wrong class gets a floor of ε/K the model may not crush to zero. ε=0 recovers the hard target.
    
        
    function smoothTarget(K, i, eps) {
  return Array.from({ length: K },
    (_, j) => (j === i ? 1 - eps : 0) + eps / K);
}
const soft = smoothTarget(K, TRUE, 0.1);
// [0.9167, 0.0167, 0.0167, 0.0167, 0.0167, 0.0167]  (sums to 1)
//   ^ true = 1 - 0.1 + 0.1/6         ^ others = 0.1/6

  5. 
        
    5
    Cross-entropy — hard vs. smoothed
    
        
    Cross-entropy is −Σ t·log p. With the hard target it collapses to −log(p_true). With the smoothed target every class contributes, so it also rewards spreading some mass around — a built-in “don’t be too peaky” penalty.
    
        
    function crossEntropy(target, p) {
  let L = 0;
  for (let j = 0; j < p.length; j++)
    if (target[j] > 0) L -= target[j] * Math.log(Math.max(p[j], 1e-12));
  return L;
}
crossEntropy(hard, p);   // = -log(p_true)          ~0.097
crossEntropy(soft, p);   // (1-e)*(-log p_true) - (e/K)*sum(log p_k)
// identity: CE(soft,p) = (1-e)*CE(onehot,p) + e*CE(uniform,p) + const

  6. 
        
    6
    The gradient p − t — and the sign flip
    
        
    The gradient of softmax+cross-entropy w.r.t. the logits is just p − t. With the hard target the true-class gradient is p_true − 1 — negative forever, always pushing the logit up. With smoothing the target is only 1−ε+ε/K, so once confidence passes that ceiling the gradient goes positive and pulls the over-confident logit back down. There is now a finite fixed point.
    
        
    const grad = p.map((pj, j) => pj - target[j]);   // dL/dz = p - t

// TRUE class, hard target:   p_true - 1        = 0.908 - 1     = -0.092  (push UP, forever)
// TRUE class, eps = 0.18:    p_true - ceiling  = 0.908 - 0.85  = +0.058  (push DOWN!)
// each WRONG class: target = eps/K > 0, so its logit can't be driven to -Infinity

    One training step — ε=0 vs. ε=0.1, side by side
    
        
    Same loss, same optimiser — only the target changes. Descend the gradient and the smoothed run stops short of extreme confidence, landing on calibrated probabilities instead of a 100% spike.
    
        
    function step(logits, target, lr) {
  const p    = softmax(logits);
  const grad = p.map((pj, j) => pj - target[j]);   // p - t
  return logits.map((z, j) => z - lr * grad[j]);   // gradient descent
}
const eps = 0.1;
const tgt = eps === 0 ? oneHot(K, TRUE) : smoothTarget(K, TRUE, eps);

let z = logits.slice();
for (let it = 0; it < 200; it++) z = step(z, tgt, 0.5);
// eps=0   -> softmax(z) collapses toward [1, 0, 0, 0, 0, 0]  (overconfident)
// eps=0.1 -> softmax(z) settles near the soft target ~0.917  (calibrated)

    The real thing — one argument in PyTorch
    
        
    You never hand-roll this in production. Frameworks fold it into the loss: pass integer targets and raw logits, set label_smoothing, done. One hyper-parameter, no architecture change, no extra compute — one of the cheapest wins in the training toolbox. Default around 0.1; tune while watching both accuracy and a calibration metric (ECE / a reliability diagram).
    
        
    import torch, torch.nn as nn

criterion = nn.CrossEntropyLoss(label_smoothing=0.1)   # <-- the whole feature

logits  = model(x)                 # raw scores, shape [batch, K]
targets = y                        # integer class indices, shape [batch]
loss    = criterion(logits, targets)   # softmax + smoothed cross-entropy, fused
loss.backward()

# family of "soft target" tricks: distillation's temperature-softened teacher,
# and Mixup / CutMix, which blend two images AND their labels into soft targets.
# eps = 0   -> plain hard cross-entropy (overconfident)
# eps ~ 0.1 -> calibrated, usually a small accuracy bump too

🎉 Day 36 of DeepLearningFromZero done.
Label smoothing replaces the hard one-hot target with a soft one — 1−ε+ε/K on the true class and a floor of ε/K on every other — because a 0/1 target is an overconfident lie: reaching softmax 1.0 demands an infinite logit, so hard cross-entropy keeps inflating confidence without end. The soft target gives a reachable, finite goal, so the gradient p−t gains a fixed point and even reverses to pull an over-confident logit back down once it passes the ceiling. The payoff is calibration — predicted probabilities that actually match how often the model is right — plus a frequent small accuracy gain, all for one hyper-parameter and zero extra compute. Mind the trade-off (too much ε makes the model under-confident) and the caveats: a smoothed teacher hurts knowledge distillation (it erases the “dark knowledge” of class similarity) and it can wash out embedding geometry needed for retrieval / metric learning. In practice it’s one argument — nn.CrossEntropyLoss(label_smoothing=0.1). 👉 Tomorrow, Day 37: Mixup & CutMix — blending two images and their labels into soft targets, taking the same “don’t be so sure” idea one step further.
