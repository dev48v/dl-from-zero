DeepLearningFromZero · Day 35

🔀 Data Augmentation — one labelled image becomes many: apply label-preserving transforms — flips, rotations, crops, brightness/contrast jitter, noise, cutout — so the model sees endless variations of the same class and learns the object instead of memorising the pixels

One image on the left. A whole training set on the right — live.

Deep nets are greedy for data, and a small dataset invites overfitting — the model memorises exact pixels instead of learning the class. Data augmentation manufactures new training examples on the fly by applying random transforms that change the pixels but not the label: a cat that is flipped, rotated, cropped, dimmed, or partly hidden is still a cat. Toggle the transforms below and drag the sliders to see one image morph; then hit Resample to watch a single picture explode into a batch of variants — exactly what the network sees, a fresh version every epoch.

🔨 Build it yourself

    
The live demo above is a from-scratch augmentation pipeline in the browser — no library, just the HTML5 canvas. We build it one transform at a time: geometric moves (flip, rotate, random crop) with the canvas transform matrix, photometric jitter (brightness, contrast) with ctx.filter, additive Gaussian noise straight on the pixel buffer, and cutout as a filled rectangle — then we fuse them into a random policy that turns one image into an endless stream, and finally map every line to what torchvision.transforms does in a real training loop.

    
    
      
  1. 
        
    1
    Two canvases: the source, and where variants land
    
        
    Draw the labelled image once onto an offscreen source canvas. Every augmentation reads from that pristine source and paints a fresh result onto the output canvas — the original is never mutated, so we can resample forever.
    
        
    // src = the one labelled example (drawn once, never touched again)
const src = document.createElement("canvas");
src.width = src.height = 180;
drawCat(src.getContext("2d"), 180, 180);   // our toy "image"

const out = document.getElementById("out");   // where the variant lands
const ctx = out.getContext("2d");
const W = out.width, H = out.height;
    
      
    2. 
      
  2. 
        
    2
    Geometric transforms — flip, rotate, all at the center
    
        
    Flips and rotations are one matrix. Translate the origin to the image center first, then rotate, then scale — a negative scale factor mirrors the axis, so scale(-1,1) is a horizontal flip and scale(1,-1) a vertical one. Doing it around the center keeps the cat in frame instead of spinning off a corner.
    
        
    function geometric(ctx, p) {
  ctx.fillStyle = "#e2e8f0";
  ctx.fillRect(0, 0, W, H);              // padding shows where the image isn't
  ctx.save();
  ctx.translate(W / 2, H / 2);           // work about the center
  ctx.rotate(p.rot * Math.PI / 180);     // degrees -> radians
  ctx.scale(p.hflip ? -1 : 1,            // negative x = mirror horizontally
            p.vflip ? -1 : 1);           // negative y = mirror vertically
  // (crop happens in the drawImage below, step 3)
  ctx.restore();
}
    
      
    3. 
      
  3. 
        
    3
    Random crop — a sub-rectangle stretched to fill
    
        
    The 9-argument drawImage takes a source rectangle (sx,sy,sw,sh) and stretches it over the whole destination. Shrink the source rectangle (zoom in) and slide it to a random offset — that is a random-resized-crop, the single most effective augmentation for natural images. The model learns to recognise the cat from just an ear or a paw.
    
        
    function drawCropped(ctx, p) {
  const sw = src.width  / p.zoom;        // zoom>1 -> smaller source window
  const sh = src.height / p.zoom;
  const sx = (src.width  - sw) * p.offX; // offX,offY in [0,1] -> random position
  const sy = (src.height - sh) * p.offY;
  // source sub-rect (sx,sy,sw,sh)  ->  full destination (-W/2..W/2)
  ctx.drawImage(src, sx, sy, sw, sh, -W / 2, -H / 2, W, H);
}
    
      
    4. 
      
  4. 
        
    4
    Photometric jitter — brightness & contrast for free
    
        
    Colour changes teach the model to ignore lighting. The canvas ships CSS-style filters: set ctx.filter before drawing and it is baked into the pixels. Brightness scales every channel; contrast pushes values away from (or toward) mid-grey. Combined with hue/saturation this is the classic ColorJitter.
    
        
    // set the filter BEFORE the drawImage so it bakes into the output
ctx.filter = `brightness(${p.brightness}) contrast(${p.contrast})`;
drawCropped(ctx, p);                 // pixels come out jittered
ctx.filter = "none";                 // reset so later steps aren't filtered

// 1.0 = unchanged; 0.5 = half as bright; 1.5 = brighter / more contrast

  5. 
        
    5
    Gaussian noise — edit the raw pixel buffer
    
        
    For per-pixel effects, pull the raw bytes with getImageData, add a Gaussian sample (Box–Muller turns two uniforms into a normal) to each RGB channel, and write them back. Sensor-like noise makes the model robust to grainy, low-light inputs and blurs the exact pixel values it might otherwise memorise.
    
        
    function addNoise(ctx, sigma) {
  const img = ctx.getImageData(0, 0, W, H);
  const d = img.data;                       // [r,g,b,a, r,g,b,a, ...]
  for (let i = 0; i < d.length; i += 4) {
    const n = gauss() * sigma;              // one sample, applied to r,g,b
    d[i]   += n; d[i+1] += n; d[i+2] += n;  // alpha (d[i+3]) untouched
  }
  ctx.putImageData(img, 0, 0);
}
function gauss() {                          // Box-Muller: uniforms -> normal
  const u = Math.random(), v = Math.random();
  return Math.sqrt(-2 * Math.log(u)) * Math.cos(2 * Math.PI * v);
}

  6. 
        
    6
    Cutout — mask a random square, keep the label
    
        
    Cutout (a.k.a. random erasing) drops a random square to a constant fill. It forces the network to spread its attention over the whole object instead of leaning on one giveaway feature — if the ears are covered, learn the whiskers. The label is unchanged: a partly hidden cat is still a cat.
    
        
    function cutout(ctx, frac) {              // frac = side length / image size
  if (frac <= 0) return;
  const s = frac * W;
  const x = Math.random() * (W - s);      // random top-left, fully inside
  const y = Math.random() * (H - s);
  ctx.fillStyle = "#6b7280";              // constant gray fill
  ctx.fillRect(x, y, s, s);
}

    7
    The policy — sample random params, one image → many
    
        
    Augmentation is a distribution, not a fixed edit. Each time we need a training example, sample fresh random parameters and run the whole pipeline. Same source, endless outputs — the model effectively trains on a dataset far larger than the one you collected, and never memorises a single frame.
    
        
    function samplePolicy() {                  // one random draw = one variant
  return {
    hflip: Math.random() < 0.5,           // coin flip
    vflip: false,                         // OFF: an upside-down cat is rare
    rot:   rand(-25, 25),                 // mild rotation
    zoom:  rand(1.0, 1.8),  offX: Math.random(), offY: Math.random(),
    brightness: rand(0.75, 1.25),
    contrast:   rand(0.8, 1.2),
    sigma:  rand(0, 22),
    cut:    Math.random() < 0.5 ? rand(0.15, 0.35) : 0,
  };
}
function augment(ctx, p) {                 // fuse all six transforms
  ctx.fillStyle = "#e2e8f0"; ctx.fillRect(0, 0, W, H);
  ctx.save();
  ctx.filter = `brightness(${p.brightness}) contrast(${p.contrast})`;
  ctx.translate(W/2, H/2);
  ctx.rotate(p.rot * Math.PI/180);
  ctx.scale(p.hflip?-1:1, p.vflip?-1:1);
  drawCropped(ctx, p);
  ctx.restore(); ctx.filter = "none";
  addNoise(ctx, p.sigma);                  // pixel-level, after drawing
  cutout(ctx, p.cut);
}
// 12 tiles = augment(tileCtx, samplePolicy()) x 12 -> the gallery above

    The real thing — and the one rule that matters
    
        
    Every canvas call maps one-to-one to a line of torchvision.transforms. The framework composes them into the training Dataset so a fresh random variant is produced on every __getitem__. The golden rule: augment the training set only. The validation/test set must stay clean — augmenting it would measure the wrong thing. (Test-time augmentation is a separate, deliberate ensembling trick.)
    
        
    import torchvision.transforms as T

train_tf = T.Compose([                       # a fresh random draw per image, per epoch
    T.RandomResizedCrop(180, scale=(0.4, 1.0)),   # step 3
    T.RandomHorizontalFlip(p=0.5),                # step 2
    T.RandomRotation(25),                         # step 2
    T.ColorJitter(brightness=0.25, contrast=0.2), # step 4
    T.ToTensor(),
    T.RandomErasing(p=0.5, scale=(0.02, 0.15)),   # step 6 (cutout)
])

# TEST/VAL: no randomness -- resize + center-crop + ToTensor ONLY
test_tf = T.Compose([T.Resize(200), T.CenterCrop(180), T.ToTensor()])

train_ds = ImageFolder("data/train", transform=train_tf)  # augmented
val_ds   = ImageFolder("data/val",   transform=test_tf)    # clean!

🎉 Day 35 of DeepLearningFromZero done.
Data augmentation fights overfitting by synthesising new training examples from the ones you have, using transforms that change the pixels but preserve the label. Geometric moves — horizontal flip, rotation, random-resized-crop — teach invariance to pose and framing; photometric jitter — brightness, contrast, colour, noise — teaches invariance to lighting and sensors; occlusion — cutout / random erasing — forces the model to use the whole object, not one shortcut feature. The catch is that label-preserving is domain-specific: a horizontal flip is fine for a cat but turns a 6 into something else and mirrors text; heavy rotation is fine for aerial photos but wrong for upright digits — choose transforms that respect what the label actually means. Because augmentation is a random policy resampled every epoch, the network effectively trains on a far larger dataset and never sees the same frame twice — which is why it generalises. And it applies to the training set only; the test set stays pristine so you measure real performance. Learned policies like AutoAugment/RandAugment, and label-mixing variants like Mixup/CutMix, take the same idea further. 👉 Tomorrow, Day 36: label smoothing — softening those hard one-hot targets to curb overconfidence.
