🪶 RMSNorm — LayerNorm minus the mean-centering and the bias: divide by the root-mean-square, rescale by γ. Fewer ops, half the params, what LLaMA and T5 use

Same job, minus the mean. RMSNorm skips the centering LayerNorm does — and lands in the same place when the input is already centered.

Both normalizers take one token’s feature vector and tame its scale. LayerNorm subtracts the mean μ, divides by the standard deviation σ, then applies a learnable gain γ and bias β. RMSNorm throws away the mean and the bias: it divides by the root-mean-square RMS(x)=√(mean(x²)+ε) and rescales by γ only. Drag the six input features (and γ, β) and read the exact arithmetic on both sides. Hit ≈ zero-mean: with a centered input the two outputs are identical, because variance equals mean-of-squares, so σ = RMS. Then add a large offset and watch LayerNorm erase it while RMSNorm lets it ride through.

🔨 Build it yourself

    
You saw that LayerNorm does two things — re-center (subtract the mean) and re-scale (divide by the spread) — and that RMSNorm keeps only the re-scaling, dividing by the root-mean-square and rescaling by γ, with no mean and no β. Here it is in code: RMSNorm from scratch in NumPy, a side-by-side check against LayerNorm (identical on zero-mean input, different once you add an offset), the LLaMA-style PyTorch module with the fp32 trick, dropping it into a pre-norm Transformer block, and finally the one-liner nn.RMSNorm.

    
    
      
  1. 
        
    1
    RMSNorm from scratch
    
        
    One statistic, no mean. Square the features, average them, take the square root (that is the RMS), divide, then scale by the learnable gain g. With g=1 the output has unit RMS.
    
        
    import numpy as np

def rmsnorm(x, g, eps=1e-6):          # x: (..., d)   g: (d,)
    ms  = np.mean(x**2, axis=-1, keepdims=True)   # mean of squares (no mean subtracted)
    rms = np.sqrt(ms + eps)                        # root-mean-square
    return g * x / rms                             # scale by gamma -- no beta

x = np.array([2.0, -1.0, 0.5, -1.5])
y = rmsnorm(x, np.ones_like(x))
print(y)                              # x divided by its RMS
print(np.sqrt((y**2).mean()))         # ~1.0  -> unit RMS when g=1

    Side by side with LayerNorm
    
        
    On an already-centered vector the two are the same (variance == mean-of-squares, so σ == RMS). Add a constant offset and LayerNorm removes it while RMSNorm keeps it — the whole difference in one experiment.
    
        
    import numpy as np
def layernorm(x, g, b, eps=1e-6):
    mu, var = x.mean(-1, keepdims=True), x.var(-1, keepdims=True)
    return g * (x - mu) / np.sqrt(var + eps) + b
def rmsnorm(x, g, eps=1e-6):
    return g * x / np.sqrt((x**2).mean(-1, keepdims=True) + eps)

d = 4; g = np.ones(d); b = np.zeros(d)
z = np.array([1.0, -0.5, 0.5, -1.0])              # already zero-mean
print(np.allclose(layernorm(z, g, b), rmsnorm(z, g), atol=1e-3))  # True

s = z + 3.0                                         # add a big offset
print(layernorm(s, g, b).round(3))                 # offset ERASED (same as z)
print(rmsnorm(s,  g).round(3))                      # offset SURVIVES -> different

    The PyTorch module (LLaMA style)
    
        
    A single learnable vector g, no bias. The one production detail: compute the RMS in float32 even for a bf16/fp16 model, then cast back — the mean-of-squares is the part that overflows or loses precision.
    
        
    import torch, torch.nn as nn

class RMSNorm(nn.Module):
    def __init__(self, d, eps=1e-6):
        super().__init__()
        self.eps = eps
        self.g = nn.Parameter(torch.ones(d))      # gain only -- no bias
    def forward(self, x):
        dt = x.dtype
        x  = x.float()                             # RMS in fp32 for stability
        rms = x.pow(2).mean(-1, keepdim=True).add(self.eps).sqrt()
        return (self.g * (x / rms)).to(dt)

y = RMSNorm(512)(torch.randn(2, 7, 512))
print(y.shape)                                     # torch.Size([2, 7, 512])

    Drop it into a pre-norm block
    
        
    Modern Transformers are pre-norm: normalize the input of each sub-layer, inside the residual branch, so the residual stream stays a clean identity path. RMSNorm is the norm plugged into that slot in essentially every recent LLM.
    
        
    import torch.nn as nn

class Block(nn.Module):
    def __init__(self, d, h):
        super().__init__()
        self.n1, self.n2 = RMSNorm(d), RMSNorm(d)
        self.attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.mlp  = nn.Sequential(nn.Linear(d, 4*d), nn.GELU(), nn.Linear(4*d, d))
    def forward(self, x):
        a, _ = self.attn(self.n1(x), self.n1(x), self.n1(x))
        x = x + a                        # pre-norm: normalize the branch input
        x = x + self.mlp(self.n2(x))     # residual path is untouched -> stable depth
        return x

    The one-liner — nn.RMSNorm
    
        
    PyTorch 2.4+ ships it. Same math as the module above, plus a functional form. This is the norm inside LLaMA, T5, Mistral and Gemma.
    
        
    import torch, torch.nn as nn, torch.nn.functional as F

norm = nn.RMSNorm(512, eps=1e-6)          # elementwise_affine=True -> learns gamma
x = torch.randn(2, 7, 512)
y = norm(x)

# functional form -- identical result:
y = F.rms_norm(x, (512,), weight=norm.weight, eps=1e-6)

# used in place of LayerNorm by LLaMA 1/2/3, T5, Mistral, Gemma, PaLM, ...

🎉 Day 31 of DeepLearningFromZero done.
LayerNorm does two jobs — re-center (subtract the mean) and re-scale (divide by the spread) — then applies a learnable γ and β. RMSNorm asks whether the re-centering earns its keep, and the answer at scale is no: it drops the mean subtraction and the bias, keeping only y = γ · x / √(mean(x²)+ε). That is one statistic instead of two, half the parameters (gain, no bias), and fewer element-wise ops — and because variance equals mean-of-squares whenever the input is already centered, it lands on the same answer as LayerNorm in the regime that dominates deep nets, while letting a genuine offset ride through when there is one. Trained models show no quality loss, which is why LLaMA, T5, Mistral, Gemma and PaLM all use RMSNorm in the pre-norm slot, with a final RMSNorm before the output head. The ε inside the square root is the one detail you cannot omit. 👉 Tomorrow, Day 32: Learning-Rate Schedules — warmup then cosine decay, the training-time knob that decides whether a Transformer converges or diverges.
