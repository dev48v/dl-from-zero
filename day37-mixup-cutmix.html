DeepLearningFromZero · Day 37

🧬 Mixup & CutMix — stop training on one image at a time: blend two training images and their labels into a soft target. Mixup ghosts them pixel-wise (λ·A + (1−λ)·B); CutMix cuts a patch of B into A and mixes the label by area — so the network learns to behave linearly between classes, which smooths decision boundaries, curbs memorisation, and improves calibration & robustness

Two images, one slider. Watch them blend — and the label blend with them.

Ordinary augmentation (Day 35) transforms one image and keeps its label. Mixup and CutMix go further: they take two images and combine them into a new training example whose label is soft — part cat, part dog — because that's honestly how much of each is now in the picture. Flip the mode and drag λ (how much 🐱 cat to keep): Mixup makes every output pixel a λ·cat + (1−λ)·dog ghost; CutMix pastes a rectangular patch of dog whose area sets the label. In both, the target underneath melts from a hard one-hot into a calibrated soft label — and that is the whole regulariser.

The loss it drives

A blended image gets a blended cross-entropy: the standard loss against each class, weighted by how much of that class is in the picture. Same “soft target” idea as Day-36 label smoothing — but the softness comes from real pixels of a second class.

Minimising this pushes the model to output the soft label — i.e. to interpolate its confidence between the two classes instead of snapping to 100%.

🔨 Build it yourself

    
The live demo runs entirely from scratch — two hand-drawn canvases and a few lines of blending, no library. We build it one line at a time: two source images with one-hot labels, a mixing weight λ from a Beta distribution, the Mixup pixel blend, the matching label blend, the CutMix patch-and-paste, the area-derived label, and the soft-label cross-entropy that ties it all together — then we map it to the few lines you actually write with timm.

    
    
      
  1. 
        
    1
    Two examples — images and their one-hot labels
    
        
    Everything operates on a pair. Grab two samples from the batch: image A with label cat, image B with label dog. Labels start as clean one-hot vectors — the softness is created by mixing, not assumed.
    
        
    const K = 2;                          // classes: [cat, dog]
const A = { img: canvasA, y: [1, 0] };   // a cat  -> one-hot cat
const B = { img: canvasB, y: [0, 1] };   // a dog  -> one-hot dog
// in a real loop B is just A shuffled: xb = xa[torch.randperm(n)]
    
      
    2. 
      
  2. 
        
    2
    Sample the mixing weight λ from Beta(α,α)
    
        
    λ is not fixed — each batch draws its own from a symmetric Beta. Small α (≈0.2) piles mass near 0 and 1, so most mixes are gentle with occasional strong ones; α=1 is uniform; large α centres on 0.5 (aggressive). It’s the one knob you tune.
    
        
    // Beta(a,a) via two Gamma draws:  g1 / (g1 + g2)
function sampleBeta(a) {
  const g = k => { let s = 0; for (let i = 0; i < k; i++) s -= Math.log(Math.random()); return s; };
  const g1 = g(Math.max(1, Math.round(a))), g2 = g(Math.max(1, Math.round(a)));
  return g1 / (g1 + g2);
}
const lam = sampleBeta(0.2);     // e.g. 0.83  (mostly image A)
    
      
    3. 
      
  3. 
        
    3
    Mixup — the pixel-wise convex blend
    
        
    Every output pixel is λ·A + (1−λ)·B. On a canvas that’s two draws with globalAlpha: paint A fully, then paint B at opacity 1−λ on top — the compositor gives exactly the convex combination.
    
        
    function mixup(out, A, B, lam) {
  const ctx = out.getContext("2d");
  ctx.globalAlpha = 1;        ctx.drawImage(A, 0, 0);   // full cat
  ctx.globalAlpha = 1 - lam;  ctx.drawImage(B, 0, 0);   // dog ghosted on top
  ctx.globalAlpha = 1;
  // result pixel = lam*A + (1-lam)*B   (a translucent overlay of both)
}
    
      
    4. 
      
  4. 
        
    4
    Blend the label the SAME way — the whole trick
    
        
    Mixing pixels but keeping a hard label would teach a lie (“this 30%-dog image is 100% cat”). So blend the label with the same λ: the target becomes λ·y_A + (1−λ)·y_B — an honest soft target, exactly the Day-36 family.
    
        
    function mixLabels(yA, yB, lam) {
  return yA.map((v, k) => lam * v + (1 - lam) * yB[k]);
}
const y = mixLabels(A.y, B.y, lam);   // lam=0.7 -> [0.7, 0.3]
//        ^ 70% cat, 30% dog  (matches the pixels you just made)
    
      
    5. 
      
  5. 
        
    5
    CutMix — cut a patch of B, paste it into A
    
        
    No ghosting: draw A, then stamp a rectangular patch of B on top. The box side is √(1−λ)·W at a random location, so its area is (1−λ) of the image. Every pixel stays a real, sharp pixel of one image.
    
        
    function cutmix(out, A, B, lam) {
  const W = out.width, H = out.height, ctx = out.getContext("2d");
  ctx.drawImage(A, 0, 0);                          // base image A
  const s = Math.sqrt(1 - lam) * W;                // patch side, area = 1-lam
  const cx = Math.random() * W, cy = Math.random() * H;
  let x = clamp(cx - s / 2, 0, W), y = clamp(cy - s / 2, 0, H);
  let x2 = clamp(x + s, 0, W),   y2 = clamp(y + s, 0, H);
  let bw = x2 - x, bh = y2 - y;                     // clipped box (may shrink at edges)
  ctx.drawImage(B, x, y, bw, bh, x, y, bw, bh);    // same-location patch of B
  return { x, y, bw, bh };
}
    
      
    6. 
      
  6. 
        
    6
    Re-derive λ from the ACTUAL pasted area
    
        
    If the box ran off the edge it got clipped, so the real patch is smaller than intended. Recompute λ from the true pasted area so the label always matches the pixels on screen — then blend labels exactly like Mixup.
    
        
    const area = (bw * bh) / (W * H);      // true fraction that is dog
const lamAdj = 1 - area;               // fraction that is still cat
const y = mixLabels(A.y, B.y, lamAdj); // e.g. patch=30% -> [0.7, 0.3]
// label ratio == patch-area fraction, honest even when the box is clipped
    
      
    7. 
      
  7. 
        
    7
    The soft-label loss — one line, both methods
    
        
    Because the target is a mix of two one-hots, the cross-entropy factorises into the ordinary loss against each class, weighted by λ. You never need a special loss — just feed the soft target (or the two hard targets with weights).
    
        
    // CE against the soft target  ==  weighted CE against each class
function mixLoss(logits, yA, yB, lam) {
  return lam * crossEntropy(logits, yA)
       + (1 - lam) * crossEntropy(logits, yB);
}
// identical to  crossEntropy(logits, mixLabels(yA, yB, lam))
// the model is pushed to output lam on cat, (1-lam) on dog
    
      
    8. 
      
  8. 
        
    8
    The real thing — a few lines with timm
    
        
    In production you configure it once and it runs per batch — timm’s Mixup even flips between Mixup and CutMix randomly and folds in label smoothing. Near-zero extra compute, no architecture change. Tune alpha and train a little longer; keep your ordinary augmentation too.
    
        
    from timm.data import Mixup

mixup_fn = Mixup(
    mixup_alpha=0.2, cutmix_alpha=1.0,   # Beta params for each
    prob=1.0, switch_prob=0.5,           # 50/50 pick Mixup vs CutMix per batch
    label_smoothing=0.1, num_classes=1000,
)

for x, y in loader:
    x, y = mixup_fn(x, y)                 # blends images AND makes soft targets
    loss = SoftTargetCrossEntropy()(model(x), y)   # soft-label CE
    loss.backward()
# family of "soft target" tricks: label smoothing, distillation, Mixup/CutMix.

🎉 Day 37 of DeepLearningFromZero done.
Mixup & CutMix train the network on images between the real ones: Mixup takes a pixel-wise convex blend λ·A + (1−λ)·B, CutMix pastes a rectangular patch of one image into the other — and in both the label is blended the same way, soft by λ (Mixup) or by patch area (CutMix). A hard 0/1 target on a mixed image would be a lie; the soft target is the honest description of what’s actually on screen, so the model is forced to interpolate linearly between classes instead of snapping to 100%. The payoff is smoother, wider decision boundaries, far less memorisation of corrupt labels, better calibration (straight from Day-36’s label smoothing), and more robustness — CutMix adds a localisation bonus because real object parts survive intact. Mind the caveats: convergence is a touch slower (train longer), and pixel-exact tasks (segmentation/detection) need box-aware variants. In practice it’s a config — timm.data.Mixup(...) plus SoftTargetCrossEntropy. 👉 Tomorrow, Day 38: Stochastic Depth — randomly dropping entire residual blocks during training, the same “drop things to generalise” idea taken to whole layers.
